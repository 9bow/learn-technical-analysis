## 레버리지와 마진 기초

암호화폐 선물 거래에서레**버리지(Leverage**)는 실제 보유 자금(마진)보다 큰 포지션을 운용할 수 있게 해주는 기능입니다.

<CandlestickDemo client:visible dataset="volatile-crypto" title="청산 캐스케이드 차트" />

$$\text{포지션 규모} = \text{마진} \times \text{레버리지}$$

```
예시:
마진: 1,000 USDT
레버리지: 10x
포지션 규모: 10,000 USDT 상당의 BTC
```

레버리지가 높을수록 적은 자본으로 큰 수익을 얻을 수 있지만, 그만큼청**산 가격이 현재가에 가까워져작**은 가격 변동에도 전체 마진을 잃을 수 있습니다.

### 마진 종류

| 마진 방식 | 설명 | 청산 리스크 |
|----------|------|-----------|
| 격리 마진 (Isolated) | 해당 포지션에만 지정한 마진 사용 | 해당 포지션의 마진만 소실 |
| 교차 마진 (Cross) | 계좌 전체 잔액을 마진으로 활용 | 계좌 전체가 청산 위험에 노출 |

## 청산 가격 계산

**청산(Liquidation**)은 포지션의 손실이 마진 전액에 도달하면 거래소가 강제로 포지션을 종료하는 것입니다. 이때 발생하는 가격 수준이청**산 가격(Liquidation Price**)입니다.

### 롱 포지션 청산 가격

$$\text{롱 청산 가격} = \text{진입가} \times \left(1 - \frac{1}{\text{레버리지}}\right)$$

```
예시: 진입가 50,000 USDT, 레버리지 10x
청산 가격 = 50,000 × (1 - 1/10) = 50,000 × 0.9 = 45,000 USDT
→ 가격이 10% 하락하면 마진 전액 소실
```

### 숏 포지션 청산 가격

$$\text{숏 청산 가격} = \text{진입가} \times \left(1 + \frac{1}{\text{레버리지}}\right)$$

```
예시: 진입가 50,000 USDT, 레버리지 10x
청산 가격 = 50,000 × (1 + 1/10) = 50,000 × 1.1 = 55,000 USDT
→ 가격이 10% 상승하면 마진 전액 소실
```

>**주의**: 실제 청산 가격은 거래소의 유지 마진율(Maintenance Margin Rate)과 수수료가 반영되어 위 공식보다 약간 다를 수 있습니다. 각 거래소의 청산 계산기를 활용하세요.

## 마진 유형별 청산 가격 상세 계산

### 격리 마진 청산 계산

격리 마진(Isolated Margin) 모드에서는 해당 포지션에 할당한 마진만이 위험에 처합니다.

```
[롱 포지션 격리 마진 상세 예시]

BTC 가격: 50,000 USDT
레버리지: 10배
할당 마진: 1,000 USDT
포지션 크기: 1,000 × 10 = 10,000 USDT = 0.2 BTC

유지 마진율 (바이낸스 기준): 0.5%
유지 마진: 10,000 × 0.5% = 50 USDT

청산 가격 (정밀 계산):
= 진입가 × (1 - 1/레버리지 + 유지 마진율)
= 50,000 × (1 - 0.1 + 0.005)
= 50,000 × 0.905
= 45,250 USDT

→ BTC가 45,250 USDT 아래로 하락하면 강제 청산
→ 진입가 대비 9.5% 하락 시 청산 (레버리지 10배)
```

```
[숏 포지션 격리 마진 상세 예시]

BTC 가격: 50,000 USDT
레버리지: 10배
할당 마진: 1,000 USDT
포지션 크기: 10,000 USDT

청산 가격:
= 진입가 × (1 + 1/레버리지 - 유지 마진율)
= 50,000 × (1 + 0.1 - 0.005)
= 50,000 × 1.095
= 54,750 USDT

→ BTC가 54,750 USDT 위로 상승하면 강제 청산
→ 진입가 대비 9.5% 상승 시 청산
```

### 교차 마진 청산 계산

교차 마진(Cross Margin) 모드에서는 계좌 전체 잔액이 마진으로 사용됩니다.

```
[교차 마진 예시]

계좌 잔액: 10,000 USDT
BTC 롱 포지션: 레버리지 10배, 노출 금액 50,000 USDT
(진입가 50,000 USDT, 1 BTC)

가용 마진: 10,000 USDT (계좌 전체)
유지 마진: 50,000 × 0.5% = 250 USDT

청산까지 허용 손실:
= 계좌 잔액 - 유지 마진
= 10,000 - 250 = 9,750 USDT

청산 가격:
= 진입가 - (허용 손실 / 포지션 수량)
= 50,000 - (9,750 / 1)
= 40,250 USDT

→ 교차 마진은 계좌 잔액 전체가 버퍼 역할
→ 청산 가격이 격리 마진보다 훨씬 아래에 위치
→ 하지만 청산 시 계좌 전액 소멸
```

:::caution
교차 마진은 청산 거리가 넓어 안전해 보이지만, 청산 **계좌 전체가** 소멸합니다. 여러 포지션을 동시에 교차 마진으로 운영하면 하나의 급변동으로 모든 포지션이 동시에 청산될 수 있습니다.
:::

### 레버리지별 청산 가격 비교표

```
진입가 50,000 USDT 롱 포지션 기준 (유지 마진율 0.5% 가정):

레버리지  청산 가격     허용 하락폭   연간 변동성 대비
2배      25,250 USDT   49.5%        매우 안전
3배      33,583 USDT   32.8%        안전
5배      40,250 USDT   19.5%        주의 필요
10배     45,250 USDT   9.5%         위험
20배     47,625 USDT   4.75%        매우 위험
50배     49,010 USDT   1.98%        극단적 위험
100배    49,505 USDT   0.99%        거의 즉시 청산
```

:::tip
비트코인은 하루 5~10% 변동이 일상적입니다. 10배 이상 레버리지는 정상적인 가격 노이즈만으로도 청산될 수 있습니다. 숙련된 트레이더도 일반적으로 3~5배 이내로 레버리지를 제한합니다.
:::

## 청산 캐스케이드 (Liquidation Cascade)

청산 캐스케이드는연**쇄 청산이**라고도 하며, 암호화폐 시장의 급등락을 증폭시키는 핵심 메커니즘입니다.

### 발생 과정

```
1. 가격이 하락 → 레버리지 롱 포지션 청산 발생
2. 청산 주문(시장가 매도)이 체결 → 추가 가격 하락
3. 더 많은 롱 포지션이 청산 가격에 도달
4. 추가 청산 → 추가 하락 (반복)
5. 청산할 포지션이 소진되거나 매수세가 유입될 때까지 지속
```

이 과정에서 가격은 단시간에수**십 퍼센트까**지 급락할 수 있으며, 반대 방향(숏 청산)에서도 동일한 급등이 발생합니다. 이를 각각롱** 스퀴즈(Long Squeeze**)**숏 스퀴즈(Short Squeeze**)라 부릅니다.

### 암호화폐 시장에서의 특수성

전통 금융 시장은 일일 가격 변동 한도(서킷 브레이커)가 있지만, 암호화폐 시장은**24시간 365일 운영되**며 가격 제한이 없습니다. 따라서 청산 캐스케이드의 충격이 훨씬 크게 나타날 수 있습니다.

## 청산 히트맵 활용

### 청산 히트맵이란?

CoinGlass 등에서 제공하는청**산 히트맵(Liquidation Heatmap**)은 현재 열려 있는 레버리지 포지션들의 청산 가격 분포를 시각화한 도구입니다 [CoinGlass, 2026].

히트맵에서 색이 진할수록(보통 빨간색/노란색**)해당 가격 구간에 청산 대기 중인 포지션이 많다는** 의미입니다.

### 히트맵 읽는 법

```
현재가 위쪽 밀집 구간 → 숏 포지션 청산 레벨 (가격 상승 시 숏 스퀴즈 발생 가능)
현재가 아래쪽 밀집 구간 → 롱 포지션 청산 레벨 (가격 하락 시 롱 스퀴즈 발생 가능)
```

시장 조성자(Market Maker) 또는 대형 세력은 이 데이터를 활용해 청산이 많이 몰린 구간으로 가격을 끌어당겨 청산을 촉발시키고 유동성을 확보하는 경향이 있습니다. 이를유**동성 사냥(Liquidity Hunt**)이라고도 합니다.

### 지지·저항으로의 활용

청산 레벨 밀집 구간은 전통적인 지지·저항 분석과 결합할 수 있습니다.

| 청산 구간 위치 | 활용 방법 |
|-------------|----------|
| 현재가 아래 대규모 롱 청산 구간 | 해당 구간이 강력한 지지로 작용 가능, 이탈 시 급락 주의 |
| 현재가 위 대규모 숏 청산 구간 | 해당 구간 도달 시 급등 가능성, 돌파 후 저항 → 지지 전환 |
| 청산 구간 공백 구간 | 가격이 빠르게 통과할 가능성 (슬리피지 위험) |

>**실전 예시**: 비트코인이 특정 가격대에서 오랫동안 횡보할 때, 그 아래에 대규모 롱 청산 구간이 형성되는 경우가 많습니다. 이 구간을 일시적으로 하방 이탈(위크아웃, Fakeout)한 뒤 청산을 소진하고 다시 회복하는 패턴이 반복됩니다.

## 청산 히트맵 상세 읽기 가이드

### 히트맵 구성 요소 해석

CoinGlass 청산 히트맵은 가격(Y축)과 시간(X축) 위에 청산 레벨 밀집도를 색상으로 표시합니다.

```
색상 강도:
노란색/흰색 (매우 밝음) → 해당 가격대에 청산 대기 포지션 매우 많음
주황색/빨간색 (중간)   → 상당한 청산 포지션 존재
파란색/어두운색         → 청산 포지션 적음

위치 해석:
현재가 위쪽 밝은 구간 → 숏 포지션 청산 레벨 (숏 스퀴즈 유발 구간)
현재가 아래 밝은 구간 → 롱 포지션 청산 레벨 (롱 스퀴즈 유발 구간)
```

### 히트맵 활용 실전 시나리오

```
[시나리오 1: 위쪽 청산 구간 활용]

현재 BTC: 50,000 USDT
히트맵: 52,000~53,000 USDT 구간에 대규모 숏 청산 레벨 집중

의미:
→ 이 구간에 다수의 숏 포지션의 청산가가 몰려 있음
→ 가격이 52,000을 돌파하면 숏 강제 청산(매수) 주문 쏟아짐
→ 추가 상승 가속 (숏 스퀴즈)

활용:
→ 50,000 지지 확인 후 롱 진입
→ 52,000~53,000 청산 구간을 1차 목표로 설정
→ 청산 구간 통과 후 모멘텀 약화 여부 확인

[시나리오 2: 아래쪽 청산 구간 활용]

현재 BTC: 50,000 USDT
히트맵: 47,000~48,000 USDT 구간에 대규모 롱 청산 레벨 집중

의미:
→ 47,000~48,000 구간 하방 이탈 시 롱 강제 청산 쏟아짐
→ 급격한 하락 가속화 가능 (롱 스퀴즈)

활용:
→ 48,000 근처에서 롱 포지션 손절 배치 주의
→ 47,500 이탈 확인 시 추가 하락 가능성 높음
→ 반대로, 47,000 구간에서 매수세 유입 후 반등 시 강력한 지지 신호
```

:::note
청산 히트맵의 데이터는 현재 열려 있는 포지션의 추정치입니다. 실제 청산 발생 여부는 시장 참여자의 마진 추가 여부에 따라 달라질 수 있으며, 히트맵 데이터만으로 확정적인 예측은 불가합니다.
:::

### 청산 구간과 기술적 지지·저항 통합

```
가장 강력한 신호: 기술적 지지·저항 + 청산 레벨 동일 구간

예시:
- 50,000 USDT = 심리적 라운드 넘버
- 50,000 USDT = 200일 이동평균
- 49,500~50,000 = 히트맵 대규모 롱 청산 구간

→ 이 구간 이탈 시 기술적 지지 붕괴 + 대규모 청산 동시 발생
→ 급격한 하락 가능성 매우 높음
→ 이 구간 위에서 매수한 롱은 손절을 49,000 아래에 배치하거나
   구간 이탈 후 재진입 전략 고려
```

## 실전 적용

### 청산 레벨을 고려한 손절 배치

```
[올바른 손절 배치 예시]

상황: BTC 50,000에서 롱 진입, 손절 계획 중

히트맵 확인 결과: 48,500~49,000에 대규모 롱 청산 구간 존재

잘못된 손절: 49,200 (청산 구간 바로 위)
→ 가격이 청산 구간을 건드리며 급락할 때 함께 청산될 위험

올바른 손절: 48,200 (청산 구간 아래)
→ 대규모 청산이 소진된 후 반등 기회를 포함
→ 또는 48,500 위에서 반등 확인 시 재진입 전략 활용
```

### 레버리지 수준 결정 기준

```
시장 상황에 따른 권장 레버리지:

횡보 (낮은 변동성):
BTC 일봉 ATR 1,000 USDT 미만 → 최대 5배 레버리지 고려

보통 변동성:
BTC 일봉 ATR 1,500~2,500 USDT → 최대 3배 레버리지

고변동성 (뉴스 이벤트, 고점/저점 근방):
BTC 일봉 ATR 3,000 USDT 이상 → 최대 2배, 또는 무레버리지

원칙: 청산 가격이 ATR × 3 이상 여유를 가져야 정상적인
      변동성에 의한 청산 위험을 줄일 수 있음
```

### 자동 청산 알림 설정

```
권장 알림 설정 (거래소 앱 기준):

1. 청산 가격의 130% 가격에서 1차 경고 알림
   (예: 청산가 45,000 → 48,500에서 알림)

2. 청산 가격의 115% 가격에서 긴급 알림
   (예: 청산가 45,000 → 51,750이 현재가일 때 = 아래쪽 10% 근접 시)

3. 알림 수신 즉시:
   - 포지션 일부 청산으로 마진율 개선
   - 또는 손절 주문 확인 및 실행
```

---

## 흔한 실수

### 1. 청산 가격을 진입 전 계산하지 않음

```
실수: "레버리지 20배로 진입하면 수익이 크겠지"
결과: 진입 후 2.5% 하락으로 청산 → 마진 전액 소멸

올바른 순서:
1. 청산 가격 계산: 진입가 × (1 - 1/레버리지 + 유지 마진율)
2. 청산 가격이 ATR × 3 이상 여유가 있는지 확인
3. 여유가 없으면 레버리지 축소 또는 진입 보류
```

### 2. 교차 마진으로 여러 포지션 동시 운영

```
위험 상황:
포지션 A (BTC 롱) + 포지션 B (ETH 롱) 모두 교차 마진
BTC와 ETH 동시 급락 시:
→ 두 포지션의 손실이 계좌 전체 잔액을 동시에 소진
→ 계좌 전액 청산

권장:
교차 마진은 단일 포지션에만 사용
여러 포지션은 격리 마진으로 각각 독립 관리
```

### 3. 대규모 청산 구간을 지지·저항으로 맹신

```
오해: "히트맵에서 48,000에 대규모 청산 구간 → 여기서 반드시 반등"
현실: 청산 구간은 지지 역할을 할 수 있지만 보장은 없음
      대규모 청산이 발생하면 오히려 급락 가속화 가능

올바른 접근:
청산 구간 = 변동성 급증 구간으로 인식
반등 확인(캔들 패턴, 거래량 급증) 후 진입
청산 구간 이탈 확인 전 포지션 진입 금지
```

### 4. 마진 추가로 청산 가격 낮추기 반복

```
잘못된 행동:
롱 포지션이 손실 중 → 청산 피하려고 마진 추가 → 청산가 낮아짐
→ 추가 하락 → 마진 추가 반복
결과: 원래 손절했어야 할 포지션에 계속 투입 → 더 큰 손실

올바른 대응:
처음 설정한 손절가를 지키고 청산 전에 손절 주문으로 포지션 정리
```

:::caution
마진 추가로 청산을 피하는 것은 본질적으로 손절을 거부하는 것과 같습니다. 결국 더 큰 손실 또는 강제 청산으로 이어지는 경우가 대부분입니다.
:::

### 5. 청산 히트맵 데이터를 100% 신뢰

```
한계:
- 히트맵은 추정 데이터, 실제 청산 레벨과 다를 수 있음
- 대형 트레이더는 OTC(장외)로 헤지하여 히트맵에 미반영
- 거래소마다 데이터 차이 존재

올바른 활용:
히트맵 = 보조 도구 (가중치 30~40%)
기술적 분석 + 온체인 데이터 + 히트맵을 종합하여 판단
```

## 고레버리지의 위험성

### 레버리지별 청산 거리 비교

| 레버리지 | 청산까지 가격 변동폭 |
|---------|------------------|
| 2x | 50% 하락 (롱 기준) |
| 5x | 20% 하락 |
| 10x | 10% 하락 |
| 25x | 4% 하락 |
| 50x | 2% 하락 |
| 100x | 1% 하락 |

비트코인은 하루에 5~10% 변동이 흔하며, 알트코인은 그 이상도 일상적입니다. 50x 이상 레버리지는하**루 안에 청산될 확률이 매우 높습니다.**
### 부분 청산과 ADL

거래소에 따라 포지션 전체가 아닌부**분 청산(Partial Liquidation**)을 먼저 시도해 마진을 확보합니다. 시장 유동성이 부족할 경우자**동 디레버리징(ADL, Auto-Deleveraging**)이 발생하여 반대 포지션 보유자의 포지션이 강제 종료될 수 있습니다.

## 핵심 요약

- 청산 가격은 `진입가 × (1 ± 1/레버리지)` 로 근사 계산하며, 레버리지가 높을수록 현재가에 가까워집니다.
- 청산 캐스케이드는 연쇄적 강제 청산으로 가격 급등락을 증폭시키는 암호화폐 시장의 고유 현상입니다.
- CoinGlass 청산 히트맵을 통해 현재 레버리지 포지션의 분포를 확인하고 청산 압력이 높은 구간을 파악할 수 있습니다.
- 청산 레벨 밀집 구간은 지지·저항으로 기능할 수 있으며, 이탈 시 급격한 가격 이동의 촉매가 됩니다.
- 고레버리지(25x 이상)는 소폭의 가격 변동에도 전액 청산이 가능하므로 신중한 리스크 관리가 필수입니다.
