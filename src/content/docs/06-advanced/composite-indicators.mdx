## 왜 지표를 결합하는가?

단일 지표는 항상 한계가 있습니다:
- **추세 추종 지표**: 횡보장에서 빈번한 거짓 신호
- **오실레이터**: 강한 추세에서 과매수/과매도 상태 지속
- **거래량 지표**: 가격 방향 없이 거래량만으로는 불완전

지표를올**바르게결**합하면 각 지표의 약점을 보완하고 신호의 신뢰도를 높일 수 있습니다.

---

## 지표 결합의 원칙

### 1. 중복 방지 (Redundancy Avoidance)

같은 유형의 지표를 여러 개 사용하는 것은중**복이**지확**인이** 아닙니다:

| 나쁜 조합 (중복) | 이유 |
|----------------|------|
| RSI + 스토캐스틱 + CCI | 모두 모멘텀 오실레이터 → 같은 정보 |
| SMA + EMA + WMA | 모두 추세 지표 → 같은 신호 |
| OBV + CMF + MFI | 모두 거래량 기반 → 같은 데이터 |

| 좋은 조합 (다차원) | 이유 |
|------------------|------|
| SMA(추세) + RSI(모멘텀) + OBV(거래량) | 3가지 다른 차원 |
| ADX(추세 강도) + 볼린저(변동성) + 스토캐스틱(모멘텀) | 3가지 다른 관점 |

### 2. 다차원 확인 (Multi-Dimensional Confirmation)

기술적 분석의 5대 차원에서 최소 3개를 결합합니다:

| 차원 | 측정 대상 | 대표 지표 |
|------|----------|----------|
|**추세**| 가격 방향 | SMA, EMA, MACD, ADX |
|**모멘텀**| 가격 변화 속도 | RSI, 스토캐스틱, CCI |
|**변동성**| 가격 변동 폭 | 볼린저 밴드, ATR |
|**거래량**| 거래 참여도 | OBV, VWAP, CMF |
|**시장 구조**| 지지/저항, 패턴 | 피보나치, 피봇 포인트 |

### 3. 시간 프레임 일관성

모든 지표가같**은 시간 프레임의** 데이터를 사용해야 합니다. 일봉 RSI와 주봉 MACD를 직접 비교하면 안 됩니다.

단, [멀티 타임프레임 분석](/technical-analysis/06-advanced/multi-timeframe)에서는 의도적으로 다른 시간 프레임을 사용하되, 각 프레임 내에서는 일관성을 유지합니다.

## 복합 매매 시스템 설계

### 시스템 1: 트리플 스크린 (Alexander Elder)

알렉산더 엘더의 3중 스크린 시스템:

**1차 스크린 (주봉 — 추세 판별**):
- 13주 EMA 방향 확인
- 상승 → 매수만 허용 / 하락 → 매도만 허용

**2차 스크린 (일봉 — 모멘텀 확인**):
- Force Index 또는 스토캐스틱 사용
- 주봉 추세 방향과 일치하는 모멘텀 신호 대기
- 상승 추세에서 오실레이터 과매도 → 매수 신호

**3차 스크린 (시간봉 — 진입 타이밍**):
- 추적 매수 주문 (Trailing Buy Stop) 사용
- 직전 고점 위에 주문 → 가격이 올라올 때 자동 진입

### 시스템 2: 추세 + 모멘텀 + 거래량

**구성 요소**:
| 역할 | 지표 | 파라미터 |
|------|------|---------|
| 추세 필터 | 50일 SMA | 가격 > SMA → 매수만 |
| 추세 강도 | ADX | ADX > 25 → 추세 존재 확인 |
| 모멘텀 | RSI(14) | 과매도(30) → 매수, 과매수(70) → 매도 |
| 거래량 확인 | OBV | OBV 상승 추세 → 매수 확인 |

**매수 조건 (모두 충족**):
1. 가격 > 50일 SMA ✓
2. ADX > 25 ✓
3. RSI가 30 이하에서 반등 ✓
4. OBV가 상승 추세 ✓

**수치 예시**:

| 조건 | Day 1 | Day 2 | Day 3 | Day 4 |
|------|-------|-------|-------|-------|
| 가격 > 50 SMA | ✓ | ✓ | ✓ | ✓ |
| ADX > 25 | ✗ (22) | ✓ (27) | ✓ (29) | ✓ (31) |
| RSI < 30 반등 | ✗ (45) | ✗ (38) | ✓ (28→32) | ✗ (42) |
| OBV 상승 | ✓ | ✓ | ✓ | ✓ |
|**신호**| — | — |**매수**| — |

→ Day 3에서 4가지 조건이 모두 충족 → 매수 진입

### 시스템 3: 볼린저 + RSI + MACD

**매수 신호**:
1. 가격이 볼린저 밴드 하단 터치 또는 이탈
2. RSI < 30 (과매도)
3. MACD 히스토그램이 음수에서 양수로 전환

**매도 신호**:
1. 가격이 볼린저 밴드 상단 터치
2. RSI > 70 (과매수)
3. MACD 히스토그램이 양수에서 음수로 전환

## 컨플루언스 존 (Confluence Zone)

여러 기술적 수준이 같은 가격대에 모이는 구간을컨**플루언스 **이라 합니다.

### 컨플루언스 요소

| 요소 | 예시 |
|------|------|
| 이동평균 | 200일 SMA = 48,500원 |
| 피보나치 | 61.8% 되돌림 = 48,200원 |
| 지지/저항 | 이전 저항선 = 48,000원 |
| 추세선 | 상승 추세선 = 48,300원 |
| 피봇 포인트 | 일간 S1 = 48,100원 |

**48,000~48,500원 구간에 5가지 지지가 수렴**→ 강력한 컨플루언스 존

### 컨플루언스 점수 시스템

각 요소에 점수를 부여하여 정량화합니다:

| 요소 | 점수 |
|------|------|
| 주요 이동평균 (50, 200일) | 2점 |
| 피보나치 주요 수준 (38.2%, 50%, 61.8%) | 2점 |
| 수평 지지/저항 (2회 이상 테스트) | 2점 |
| 추세선 | 1점 |
| 보조 이동평균 (20일 등) | 1점 |
| 피봇 포인트 | 1점 |
| 볼린저 밴드 | 1점 |
| 피보나치 보조 수준 (23.6%, 78.6%) | 1점 |

**점수 해석**:
- **7점 이상**: 매우 강한 컨플루언스 → 적극적 포지션
- **5~6**: 강한 컨플루언스 → 표준 포지션
- **3~4**: 보통 → 소량 포지션
- **2점 이하**: 약한 → 진입 보류

## 신호 강도 기반 포지션 사이징

복합 지표의 신호 강도에 따라 [포지션 크기](/technical-analysis/07-risk-management/position-sizing)를 조절합니다:

### 점수 기반 사이징

$$\text{포지션 비중} = \text{기본 비중} \times \frac{\text{신호 점수}}{\text{최대 점수}}$$

**예시**: 기본 비중 = 자본의 10%, 최대 점수 = 10

| 시나리오 | 신호 점수 | 포지션 비중 |
|---------|----------|------------|
| 추세+모멘텀+거래량+구조 일치 | 9/10 | 9% |
| 추세+모멘텀 일치, 거래량 미확인 | 6/10 | 6% |
| 추세만 일치 | 3/10 | 3% |

### 확신도 레벨

| 레벨 | 조건 | 포지션 | [손절](/technical-analysis/07-risk-management/stop-loss) |
|------|------|--------|-------|
| A (높음) | 4/4 차원 일치 | 풀 사이즈 | 타이트 |
| B (보통) | 3/4 차원 일치 | 2/3 사이즈 | 보통 |
| C (낮음) | 2/4 차원 일치 | 1/3 사이즈 | 와이드 |
| D (없음) | 1/4 이하 | 진입 안 함 | — |

## 복합 지표의 함정

### 분석 마비 (Analysis Paralysis)

지표를 너무 많이 사용하면:
- 서로 상충하는 신호 발생
- 진입 기회를 놓침
- 의사결정 지연

**해결**: 최대 3~4개 지표로 제한. 각 지표가다**른 차원을** 커버하는지 확인.

### 후행성 누적

모든 기술적 지표는 후행적입니다. 여러 지표를 결합하면 후행성이누**적됩**니다:

- SMA 20일(후행 ~10일) + MACD(후행 ~13일) = 진입이 늦어짐
- **해결**: 최소 하나의 선행/동행 요소 포함 (가격 행동, 거래량 급증)

### 곡선 피팅 (Curve Fitting)

과거 데이터에 완벽히 맞는 복합 규칙을 만드는 것:
- "RSI < 28.5이고 볼린저 %B < 0.12이고 MACD 히스토그램 < -0.003일 때만 매수"
- → 과최적화, [백테스팅](/technical-analysis/06-advanced/backtesting-basics)에서만 좋은 성과

**해결**: 파라미터는 표준값 사용 (RSI 14, SMA 20/50/200 등). 소수점 최적화 금지.

## 흔한 실수

### 1. 같은 유형의 지표 중복 사용
RSI + 스토캐스틱 + CCI를 모두 쓰면 "3중 확인"이 아니라 "같은 말 3번"입니다.

### 2. 모든 지표가 일치할 때만 진입
완벽한 일치를 기다리면 거래 기회가 거의 없습니다. 핵심 2~3개 일치 + 보조 1~2개 중립 수준이면 충분합니다.

### 3. 지표 파라미터 과최적화
"RSI 13이 RSI 14보다 백테스트 성과가 0.3% 좋다"는 이유로 비표준 파라미터를 사용하는 것은 과최적화입니다.

### 4. 시장 체제 무시
횡보장에서는 오실레이터 중심, 추세장에서는 추세 지표 중심으로가**중치를 조절해**야 합니다. 고정된 복합 시스템은 모든 장세에 맞지 않습니다.
