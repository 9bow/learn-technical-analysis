## 백테스팅이란?

백테스팅(Backtesting)은 트레이딩 전략을과**거 데이터에 적용하여 성과를 시뮬레이션하**는 과정입니다. "이 전략이 과거에 작동했는가?"를 객관적으로 검증합니다.

### 왜 필요한가?

-**주관적 판단 제거**: "느낌"이 아닌 데이터 기반 의사결정
-**전략 비교**: 여러 전략의 성과를 동일 조건에서 비교
-**리스크 사전 파악**: 최대 낙폭, 연속 손실 횟수 등을 미리 확인
-**파라미터 최적화**: 이동평균 기간, RSI 기준값 등의 최적값 탐색

### 한계

- 과거 성과가 미래를 보장하지 않음
- 시장 구조 변화(체제 전환)를 반영하지 못함
- 거래 비용, 슬리피지, 유동성을 정확히 모델링하기 어려움
- 과최적화 위험

## 백테스팅 프로세스

### 1단계: 전략 정의

모든 매매 규칙을명**확하고 기계적으**로 정의합니다:

```
전략: 골든 크로스 / 데드 크로스
- 진입 (매수): 20일 SMA가 50일 SMA를 상향 돌파
- 청산 (매도): 20일 SMA가 50일 SMA를 하향 돌파
- 포지션 크기: 자본의 100% (단순화)
- 손절: 없음 (시스템 청산만 사용)
```

### 2단계: 데이터 준비

| 항목 | 최소 요건 | 권장 |
|------|----------|------|
| 기간 | 최소 2년 | 5~10년 이상 |
| 데이터 유형 | OHLCV | OHLCV + 거래량 |
| 주기 | 전략에 맞는 봉 (일봉, 시간봉) | 다중 주기 검증 |
| 품질 | 수정주가 사용 | 배당, 액면분할 반영 |

### 3단계: 시뮬레이션 실행

각 봉마다 전략 규칙을 적용:

1. 현재 포지션 상태 확인 (매수/미보유)
2. 진입/청산 조건 평가
3. 조건 충족 시 거래 실행 (다음 봉 시가 기준)
4. 포지션 가치 및 계좌 잔고 업데이트
5. 거래 기록 저장

### 4단계: 성과 분석

## 주요 성과 지표

### 1. 총 수익률 (Total Return)

$$R_{\text{total}} = \frac{V_{\text{final}} - V_{\text{initial}}}{V_{\text{initial}}} \times 100\%$$

### 2. 연환산 수익률 (CAGR)

$$\text{CAGR} = \left(\frac{V_{\text{final}}}{V_{\text{initial}}}\right)^{\frac{1}{n}} - 1$$

여기서 $n$은 투자 기간(년).

### 3. 최대 낙폭 (Maximum Drawdown)

$$\text{MDD} = \min_{t} \left(\frac{V_t - V_{\text{peak}}}{V_{\text{peak}}}\right)$$

누적 수익 곡선에서 고점 대비 최대 하락 폭. 전략의최**악의 시나리오를** 보여줍니다.

**수치 예시**:

| 시점 | 계좌 잔고 | 고점 | 낙폭 |
|------|----------|------|------|
| T1 | 10,000 | 10,000 | 0% |
| T2 | 12,000 | 12,000 | 0% |
| T3 | 10,800 | 12,000 | -10.0% |
| T4 | 9,600 | 12,000 |**-20.0%**|
| T5 | 11,400 | 12,000 | -5.0% |
| T6 | 13,000 | 13,000 | 0% |

→ MDD =**-20.0%**(T4 시점)

### 4. 샤프 비율 (Sharpe Ratio)

$$S = \frac{R_p - R_f}{\sigma_p}$$

- $R_p$: 전략의 연환산 수익률
- $R_f$: 무위험 수익률 (예: 국채 금리)
- $\sigma_p$: 전략 수익률의 연환산 표준편차

| 샤프 비율 | 해석 |
|----------|------|
| < 0 | 무위험 수익률보다 낮음 |
| 0 ~ 1.0 | 보통 |
| 1.0 ~ 2.0 | 좋음 |
| 2.0 ~ 3.0 | 매우 좋음 |
| > 3.0 | 의심스러움 (과최적화 가능성) |

**수치 예시**:
- 전략 연환산 수익률: 15%
- 무위험 수익률: 3%
- 수익률 표준편차: 18%

$$S = \frac{0.15 - 0.03}{0.18} = \frac{0.12}{0.18} = 0.67$$

→ 보통 수준. 리스크 대비 수익이 탁월하지는 않음.

### 5. 소르티노 비율 (Sortino Ratio)

$$\text{Sortino} = \frac{R_p - R_f}{\sigma_d}$$

샤프 비율과 유사하지만, $\sigma_d$**하방 변동성만** 사용합니다. 상방 변동성(수익)은 패널티를 주지 않아 더 합리적인 측정입니다.

### 6. 승률과 손익비

$$\text{승률} = \frac{\text{수익 거래 수}}{\text{전체 거래 수}} \times 100\%$$

$$\text{손익비 (Profit Factor)} = \frac{\text{총 수익}}{\text{총 손실}}$$

$$\text{기대값} = (\text{승률} \times \text{평균 수익}) - ((1 - \text{승률}) \times \text{평균 손실})$$

**수치 예시**:
- 100회 거래, 40회 수익, 60회 손실
- 평균 수익: 5만원, 평균 손실: 2만원

$$\text{승률} = 40\% $$

$$\text{기대값} = (0.4 \times 50{,}000) - (0.6 \times 20{,}000) = 20{,}000 - 12{,}000 = 8{,}000\text{원}$$

→ 승률은 40%이지만 양의 기대값 → 장기적으로 수익성 있는 전략

### 7. 칼마 비율 (Calmar Ratio)

$$\text{Calmar} = \frac{\text{CAGR}}{|\text{MDD}|}$$

수익률 대비 최대 낙폭의 비율. 1 이상이면 양호.

## 거래 비용 모델링

백테스팅에서 거래 비용을 무시하면 실제 성과와 큰 괴리가 발생합니다.

### 포함해야 할 비용

| 비용 항목 | 설명 | 일반적 범위 |
|----------|------|------------|
| 수수료 | 브로커 거래 수수료 | 0.01~0.3% |
| 슬리피지 | 주문가와 체결가의 차이 | 0.05~0.5% |
| 스프레드 | 매수/매도 호가 차이 | 0.01~1% |
| 시장 충격 | 대량 주문이 가격에 미치는 영향 | 규모에 비례 |

### 비용이 성과에 미치는 영향

**예시**: 연 200회 거래하는 단기 전략

- 거래당 총 비용: 0.2% (수수료 + 슬리피지)
- 연간 비용: $200 \times 0.2\% \times 2\text{(왕복)} = 80\%$

→ 연 80%의 수수료 부담! 이 경우 총 수익률이 80%를 넘지 않으면 손실

**교훈**: 거래 빈도가 높을수록 비용 모델링이 중요

## 과최적화 (Overfitting)

### 정의

전략이과**거 데이터의 노이즈까지 학습하**여, 과거에는 완벽하지만 미래에는 작동하지 않는 상태.

### 과최적화 징후

1.**파라미터 수 > 거래 **: 10개의 파라미터로 100회 거래를 최적화 → 위험
2.**극단적 성과**: 연 100%+ 수익, 승률 90%+ → 의심
3.**파라미터 민감도**: 파라미터를 조금만 바꿔도 성과가 급변
4.**인샘플/아웃오브샘플 괴리**: 학습 데이터에서만 좋은 성과

### 파라미터 안정성 테스트

최적 파라미터 주변의 성과를 확인합니다:

| SMA 단기/장기 | 연 수익률 | 샤프 비율 |
|--------------|----------|----------|
| 18/48 | 12.1% | 0.85 |
| 19/49 | 11.8% | 0.82 |
|**20/5**0|**12.5%**|**0.8**8|
| 21/51 | 12.0% | 0.84 |
| 22/52 | 11.5% | 0.79 |

→ 주변 파라미터에서도 비슷한 성과 =**강건(robust)한 전략**
반대로:

| RSI 기간/기준값 | 연 수익률 | 샤프 비율 |
|---------------|----------|----------|
| 13/29 | 3.2% | 0.15 |
| 14/30 | 25.6% | 2.10 |
| 15/31 | 2.8% | 0.12 |

→ 특정 파라미터에서만 성과 급등 =**과최적화 가능성 높음**

## 검증 방법론

### 1. 인샘플 / 아웃오브샘플 분할

```
전체 데이터: 2015-01 ~ 2025-12 (11년)
├── 인샘플 (학습): 2015-01 ~ 2021-12 (7년, 64%)
└── 아웃오브샘플 (검증): 2022-01 ~ 2025-12 (4년, 36%)
```

- 인샘플에서 전략 개발 및 최적화
- 아웃오브샘플에서 성과 확인 (한 번만!)
- 아웃오브샘플로 재최적화하면 의미 없음

### 2. 워크포워드 분석 (Walk-Forward Analysis)

시간 순서대로 학습/검증을 반복합니다:

```
윈도우 1: 학습 [2015-2018] → 검증 [2019]
윈도우 2: 학습 [2016-2019] → 검증 [2020]
윈도우 3: 학습 [2017-2020] → 검증 [2021]
윈도우 4: 학습 [2018-2021] → 검증 [2022]
윈도우 5: 학습 [2019-2022] → 검증 [2023]
```

각 검증 기간의 성과를 결합하여 전체 아웃오브샘플 성과를 추정합니다.

**워크포워드 효율 (WFE**):

$$\text{WFE} = \frac{\text{아웃오브샘플 CAGR}}{\text{인샘플 CAGR}} \times 100\%$$

- WFE > 50%: 양호
- WFE > 70%: 우수
- WFE < 30%: 과최적화 의심

### 3. 몬테카를로 시뮬레이션

거래 순서를 무작위로 섞어 수천 번 시뮬레이션:

1. 원래 거래 결과 목록을 무작위 순서로 재배열
2. 누적 수익 곡선을 다시 계산
3. 1,000~10,000회 반복
4. MDD, 최종 자본 등의 분포를 확인

→ "운이 좋아서 성과가 좋았을 확률"을 정량화

## 실전 백테스팅 예시

### 골든/데드 크로스 전략 백테스팅

**전략 규칙**:
- 매수: 20일 SMA > 50일 SMA (골든 크로스)
- 매도: 20일 SMA < 50일 SMA (데드 크로스)
- 초기 자본: 10,000,000원
- 거래 비용: 편도 0.15%

**가상 결과 (5년**):

| 지표 | 값 |
|------|-----|
| 총 수익률 | 45.2% |
| CAGR | 7.7% |
| MDD | -18.3% |
| 샤프 비율 | 0.62 |
| 칼마 비율 | 0.42 |
| 총 거래 수 | 34회 |
| 승률 | 38.2% |
| 평균 수익 거래 | +8.5% |
| 평균 손실 거래 | -2.1% |
| 손익비 | 2.49 |
| 최대 연속 손실 | 5회 |

**분석**:
- 승률은 38%로 낮지만, 손익비(4:1)가 높아 양의 기대값
- MDD -18.3%는 감내 가능한 수준
- 샤프 비율 0.62는 보통 → 필터 추가로 개선 여지
- 거래 빈도 낮음 (연 7회) → 거래 비용 영향 미미

## 흔한 실수

### 1. 미래 정보 사용 (Look-Ahead Bias)
현재 시점에서 알 수 없는 미래 데이터를 전략에 사용. 예: "이 달의 최고가에서 매도"는 월말에야 알 수 있음.

### 2. 생존자 편향 (Survivorship Bias)
상장폐지된 종목을 제외하고 백테스팅하면 성과가 과대평가됨. 2000년 IT 버블 당시 상장폐지된 수백 개 종목이 빠지면 당시 전략 성과가 왜곡됨.

### 3. 데이터 스누핑 (Data Snooping)
같은 데이터에서 수백 개의 전략을 테스트하면, 우연히 좋은 성과를 보이는 전략이 반드시 나타남. 다중 검정 보정(Bonferroni 등)이 필요.

### 4. 거래 비용 무시
특히 단기 전략에서 치명적. 위의 거래 비용 섹션 참조.

### 5. 유동성 가정
소형주나 거래량이 적은 종목에서 "원하는 가격에 원하는 수량을 즉시 체결"할 수 있다는 비현실적 가정.
