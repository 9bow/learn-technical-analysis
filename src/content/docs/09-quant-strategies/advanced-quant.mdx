## 전략 24: 페어 트레이딩 (Statistical Arbitrage)

**페어 트레이딩(Pair Trading**)은 공적분(Cointegration) 관계에 있는 두 자산 간의 스프레드가 역사적 평균으로 회귀하는 특성을 이용하는 통계적 차익거래 전략입니다. 단순 방향성 베팅이 아닌, 두 자산의 상대적 가격 차이를 거래합니다.

### 핵심 원리: 공적분과 스프레드

두 자산 A, B가 장기적으로 함께 움직이는 공적분 관계일 때, 스프레드는 일정 범위 내에서 진동합니다.

$$\text{Spread} = P_A - \beta \times P_B$$

$$\beta = \frac{\text{Cov}(P_A, P_B)}{\text{Var}(P_B)}$$

$$Z_{spread} = \frac{\text{Spread} - \mu_{spread}}{\sigma_{spread}}$$

- $P_A$, $P_B$: 두 자산의 현재 가격
- $\beta$: 헤지 비율 (최소분산 회귀로 산출)
- $\mu_{spread}$: 스프레드 이동평균
- $\sigma_{spread}$: 스프레드 이동 표준편차
- $Z_{spread}$: Z-점수 (현재 스프레드가 평균에서 몇 표준편차 벗어났는지)

### 진입 및 청산 기준

| Z-점수 | 신호 | 행동 |
|--------|------|------|
| Z > +2 | 스프레드 과도하게 확대 | A 매도 + B 매수 (스프레드 축소 기대) |
| Z < -2 | 스프레드 과도하게 축소 | A 매수 + B 매도 (스프레드 확대 기대) |
| -0.5 < Z < +0.5 | 평균 회귀 완료 | 포지션 청산 |
| \|Z\| > +3 | 극단적 이탈 | 추가 매수 또는 손절 검토 |

### 공적분 검정 + 거래 의사코드

```
[1단계: 공적분 관계 검증]

함수 check_cointegration(가격_A, 가격_B):
    # Engle-Granger 2단계 검정
    beta = 선형회귀(가격_A ~ 가격_B).기울기
    spread = 가격_A - beta × 가격_B

    # ADF 검정으로 스프레드 정상성 확인
    adf_결과 = ADF_검정(spread)

    만약 adf_결과.p값 < 0.05:
        반환 True, beta  # 공적분 관계 확인됨
    그렇지 않으면:
        반환 False, None

[2단계: Z-점수 계산]

함수 계산_Z점수(spread, 기간=60):
    rolling_평균 = spread.이동평균(기간)
    rolling_표준편차 = spread.이동표준편차(기간)
    Z = (spread - rolling_평균) / rolling_표준편차
    반환 Z

[3단계: 거래 실행]

매일 반복:
    공적분_여부, beta = check_cointegration(가격_A[최근90일], 가격_B[최근90일])

    만약 공적분_여부 == False:
        포지션 전량 청산  # 관계 붕괴 — 전략 중단
        계속

    spread = 가격_A[현재] - beta × 가격_B[현재]
    Z = 계산_Z점수(spread)

    만약 포지션 없음:
        만약 Z > +2.0:
            A 매도, B 매수 (각각 동일 금액)
        만약 Z < -2.0:
            A 매수, B 매도 (각각 동일 금액)

    만약 포지션 있음:
        만약 -0.5 < Z < +0.5:
            전체 청산 (수익 실현)
        만약 |Z| > 3.5:
            전체 청산 (관계 붕괴 손절)
```

### 실전 적용

**코스피 관련주 페어**
- 현대차 / 기아: 동일 그룹사, 글로벌 완성차 수요에 동반 반응
- KB금융 / 신한지주: 국내 금리 환경에 동조화 움직임
- 삼성전자 / SK하이닉스: 반도체 업황에 함께 민감

**암호화폐 시장**
- BTC/ETH: 시장 전반 방향성에 동조하나 비율이 변동
- BTC 현물 / BTC 선물: 베이시스 수렴을 이용한 캐리 트레이드
- 동일 프로젝트의 거래소 간 가격 차이

```
[코스피 예시 — 현대차/기아]

기간: 2023-01-01 ~ 2023-12-31 (일봉 데이터)

ADF 검정 결과: p값 = 0.018 → 공적분 관계 확인
beta = 0.72 (현대차 = 0.72 × 기아 + 잔차)

Z > +2 진입 발생: 연간 11회
평균 보유 기간: 8거래일
승률: 72.7% (8승 3패)
평균 수익: +1.8% / 평균 손실: -2.3%
```

### 장단점

**장점**
- 시장 방향 중립 전략 — 전체 하락장에서도 수익 가능
- 롱/숏 동시 포지션으로 시장 리스크 헤지
- 통계적 기반의 명확한 진입·청산 기준

**단점**
- **공적분 관계 붕괴 위험**: 두 기업의 펀더멘털 변화(M&A, 사업 분리)로 관계가 영구적으로 깨질 수 있음
- **거래소 리스크**: 두 자산을 동시에 거래할 수 있는 환경 필요 (양방향 주문, 공매도 가능 계좌)
- **레버리지 위험**: 스프레드가 예상과 반대로 지속 확대되면 마진콜 가능
- **룩어헤드 바이어스**: 백테스트 시 공적분 검정에 미래 데이터 혼입 주의

:::caution
공적분 관계는 과거에 존재했더라도 미래를 보장하지 않습니다. 실제 운용 시 매월 또는 분기별로 공적분 검정을 재실행하고, 관계 붕괴 시 즉시 청산하는 규칙을 반드시 포함하세요.
:::

## 전략 25: 마켓 메이킹 기초 (Market Making Basics)

**마켓 메이킹(Market Making**)은 호가창의 양방향(매수·매도)에 지정가 주문을 동시에 제시하여 스프레드 수익을 반복적으로 획득하는 전략입니다. 유동성 공급자로서 거래 상대방이 되는 역할을 합니다.

### 핵심 원리: 스프레드 수익과 인벤토리 관리

$$\text{Bid} = \text{MidPrice} - \frac{spread}{2}$$

$$\text{Ask} = \text{MidPrice} + \frac{spread}{2}$$

$$\text{PnL} = (\text{Ask} - \text{Bid}) \times \text{체결 횟수} - \text{인벤토리 손실}$$

인벤토리를 한 방향으로 계속 보유하면 가격 방향 리스크에 노출됩니다. 이를 정량화한 것이 인벤토리 리스크입니다.

$$\text{Inventory Risk} = \gamma \times \sigma^2 \times T \times q$$

- $\gamma$: 리스크 회피 계수 (운용자 설정)
- $\sigma^2$: 자산 가격 분산
- $T$: 남은 거래 시간
- $q$: 현재 보유 인벤토리 수량 (음수 = 숏)

### Avellaneda-Stoikov 모델 개요

학계에서 가장 널리 인용되는 마켓 메이킹 이론 모델입니다. 인벤토리 리스크를 반영하여 중간 가격(reservation price)을 조정하고, 최적 호가를 산출합니다.

$$r = s - q \cdot \gamma \cdot \sigma^2 \cdot (T - t)$$

$$\delta^{bid} + \delta^{ask} = \gamma \cdot \sigma^2 \cdot (T - t) + \frac{2}{\gamma} \ln\left(1 + \frac{\gamma}{\kappa}\right)$$

- $r$: 인벤토리 조정 후 예약 가격 (reservation price)
- $s$: 현재 중간 가격
- $\kappa$: 주문 도달률 파라미터

실제 구현은 복잡하지만 핵심 직관은 간단합니다. **인벤토리가 쌓이면 해당 방향의 호가를 불리하게 이동시켜 신규 인벤토리 누적을 억제합니다.**

### 인벤토리 관리 전략

```
[인벤토리 누적 시 호가 조정 예시]

기준 상태: BTC 인벤토리 0 (중립)
Bid: 50,000 / Ask: 50,020 (스프레드 20 USDT)

매수 편향 발생: +2 BTC 보유
→ 매수 측 노출 증가 (추가 매수 원치 않음)
조정 후 Bid: 49,980 / Ask: 50,000
→ Bid를 낮춰 추가 매수 억제
→ Ask를 낮춰 매도 체결 유도 (인벤토리 감소)

매도 편향 발생: -2 BTC 보유 (숏)
→ 매도 측 노출 증가
조정 후 Bid: 50,020 / Ask: 50,040
→ Bid를 올려 매수 체결 유도 (숏 커버)
→ Ask를 올려 추가 매도 억제
```

### 마켓 메이킹 의사코드

```
[단순화된 마켓 메이킹 루프]

초기 설정:
    목표_스프레드 = 0.02%    # 총 스프레드
    최대_인벤토리 = 5 BTC   # 한 방향 최대 보유량
    인벤토리_조정_계수 = 0.005% per BTC

매 틱(또는 N초)마다:
    중간가격 = (Best_Ask + Best_Bid) / 2
    인벤토리 = 현재_보유_BTC  # 양수=롱, 음수=숏

    # 인벤토리에 따른 예약 가격 조정
    예약가격 = 중간가격 - 인벤토리 × 인벤토리_조정_계수 × 중간가격

    # 호가 계산
    내_Bid = 예약가격 × (1 - 목표_스프레드 / 2)
    내_Ask = 예약가격 × (1 + 목표_스프레드 / 2)

    # 기존 주문 취소 후 새 주문 제출
    기존_주문 전량 취소()
    지정가_매수주문(내_Bid, 수량=0.1 BTC)
    지정가_매도주문(내_Ask, 수량=0.1 BTC)

    # 인벤토리 한도 초과 시 강제 청산
    만약 |인벤토리| > 최대_인벤토리:
        시장가_청산(인벤토리 / 2)  # 절반 시장가 청산
```

### 실전 고려사항

**수익 조건**
- 체결 수수료가 스프레드보다 낮아야 함 (메이커 수수료 구조 필수)
- 일정 거래량 이상의 유동성이 있는 시장에서만 작동
- 단방향 급등·급락 시 인벤토리 손실이 스프레드 수익을 초과할 수 있음

### 장단점

**장점**
- 방향성 무관 — 가격이 오르내려도 스프레드 수익 발생
- 거래 횟수가 많을수록 수익이 누적되는 구조
- 거래소 메이커 수수료 리베이트 혜택

**단점**
- **고빈도 인프라 필요**: 초저지연 서버, 거래소 코로케이션 환경 없이는 경쟁 불가
- **급등락 리스크**: 강한 방향성 움직임에서 인벤토리 손실 급증
- **경쟁 심화**: 전문 마켓 메이킹 회사와의 직접 경쟁 — 개인 투자자에게 불리
- **거래소 API 속도 제한**: 빈번한 주문 취소·재제출에 API 한도 초과 위험

:::tip
개인 투자자가 정식 마켓 메이킹을 시도하는 것은 현실적으로 매우 어렵습니다. 그러나 마켓 메이킹의 원리(스프레드 수익, 인벤토리 리스크)를 이해하면 지정가 주문 활용 전략과 유동성 분석 능력이 크게 향상됩니다.
:::

## 전략 26: ML 시그널 앙상블 개요 (Conceptual Overview)

**ML 앙상블(ML Ensemble**)은 여러 머신러닝 모델이 생성한 매매 신호를 가중 결합하여 단일 모델보다 안정적인 예측을 만드는 방식입니다. 어떤 하나의 모델도 시장을 완벽히 예측할 수 없기 때문에, 다양한 관점을 결합하는 것이 핵심입니다.

### 앙상블 신호 결합 공식

다수결 방식의 가중 앙상블:

$$\text{Signal} = \text{sign}\left(\sum_{i=1}^{N} w_i \cdot s_i\right)$$

- $N$: 모델 수
- $w_i$: 모델 $i$의 가중치 (검증 성능 기반으로 설정)
- $s_i$: 모델 $i$의 신호 (+1 = 매수, -1 = 매도, 0 = 중립)
- $\text{sign}(\cdot)$: 최종 신호 방향 결정

$$\text{확신도} = \left|\frac{\sum_{i=1}^{N} w_i \cdot s_i}{\sum_{i=1}^{N} w_i}\right|$$

확신도가 0.5 이상일 때만 실제 거래를 실행하고, 그 미만이면 관망하는 필터를 추가하면 오신호를 줄일 수 있습니다.

### 특성(Feature) 선택

모델 입력에 사용하는 대표적 기술적 특성:

| 카테고리 | 특성 | 계산 방법 |
|---------|------|---------|
| 모멘텀 | RSI(14) | 14일 상대강도지수 |
| 추세 | MACD 히스토그램 | EMA(12) - EMA(26) - Signal(9) |
| 변동성 | 볼린저 밴드 %B | (현재가 - 하단) / (상단 - 하단) |
| 거래량 | 거래량 비율 | 현재 거래량 / 20일 평균 거래량 |
| 추세 강도 | ADX(14) | 14일 평균 방향성 지수 |
| 가격 위치 | 52주 고저 %위치 | (현재가 - 52주 저점) / (52주 고점 - 저점) |

### 주요 모델 종류

**로지스틱 회귀 (Logistic Regression)**
- 가장 단순하고 해석 가능한 모델
- 각 특성의 기여도를 계수로 확인 가능
- 과적합 위험 낮음, 비선형 패턴 포착 어려움
- 기준 모델(Baseline)로 먼저 검증

**랜덤 포레스트 (Random Forest)**
- 여러 결정 트리의 결합으로 과적합 억제
- 특성 중요도(Feature Importance) 산출 가능
- 하이퍼파라미터 튜닝 상대적으로 간단
- 중소형 데이터셋에서 안정적 성능

**LSTM (Long Short-Term Memory)**
- 시계열 데이터의 장기 의존성 학습
- 가격 패턴의 순서 정보를 활용
- 대량 데이터와 GPU 환경 필요
- 과적합 위험 높음, 해석 어려움

### 개념 수준 의사코드

```
[앙상블 구성 — 단순화된 개념]

[학습 단계]

함수 학습_앙상블(과거_데이터):
    # 특성 생성
    특성 = [
        RSI(14),
        MACD_히스토그램(),
        볼린저_퍼센트B(),
        거래량_비율(20),
        ADX(14)
    ]

    # 레이블: 다음 N일 수익률 양수 → 1(매수), 음수 → -1(매도)
    레이블 = sign(미래_N일_수익률)

    # 모델 학습 (시계열 분할 교차검증 사용)
    모델1 = 로지스틱회귀.학습(특성, 레이블)
    모델2 = 랜덤포레스트.학습(특성, 레이블)
    모델3 = LSTM.학습(특성, 레이블)

    # 검증 성능 기반 가중치 설정
    w1 = 검증_샤프비율(모델1)
    w2 = 검증_샤프비율(모델2)
    w3 = 검증_샤프비율(모델3)

    반환 [모델1, 모델2, 모델3], [w1, w2, w3]

[실시간 예측 단계]

함수 앙상블_신호(현재_특성, 모델들, 가중치):
    s1 = 모델1.예측(현재_특성)   # +1 또는 -1
    s2 = 모델2.예측(현재_특성)
    s3 = 모델3.예측(현재_특성)

    결합_신호 = w1×s1 + w2×s2 + w3×s3
    확신도 = |결합_신호| / (w1 + w2 + w3)

    만약 확신도 >= 0.5:
        반환 sign(결합_신호)  # +1 또는 -1
    그렇지 않으면:
        반환 0  # 중립, 거래 없음
```

### 과적합 방지 체크리스트

실제 ML 트레이딩 시스템에서 가장 흔한 실패 원인은 과적합입니다.

```
[과적합 방지 필수 점검]

1. 시계열 분할 교차검증 사용
   → 미래 데이터를 학습에 사용하는 룩어헤드 바이어스 방지
   → Walk-forward 검증 권장

2. 충분한 아웃오브샘플(OOS) 데이터 확보
   → 전체 데이터의 최소 30%를 학습에 사용하지 않음
   → OOS 성능이 인샘플의 70% 이상이어야 신뢰 가능

3. 특성 수를 샘플 수에 비례하여 제한
   → 특성이 너무 많으면 과적합 가능성 증가
   → 일반적으로 특성 수 < 샘플 수 / 10

4. 거래 비용 반영
   → 신호가 아무리 좋아도 수수료 + 슬리피지 차감 후 수익성 확인

5. 통계적 유의성 검증
   → 성과가 운에 의한 것이 아님을 확인 (p-value < 0.05)
```

### 장단점

**장점**
- 단일 모델보다 안정적인 예측 (개별 오류 상쇄)
- 다양한 시장 국면에 대응 가능
- 지속적 재학습으로 시장 변화 적응 가능

**단점**
- **과적합 위험**: 역사적 데이터에 지나치게 최적화되어 실전에서 성능 저하
- **데이터 품질 의존성**: 잘못된 데이터(생존 편향, 결측값)는 모델 전체를 오염
- **구현 복잡성**: 데이터 파이프라인, 모델 관리, 실시간 특성 계산 인프라 필요
- **해석 어려움**: 왜 이 신호를 냈는지 설명하기 어려운 블랙박스 문제
- **시장 체제 변화**: 학습 시기와 다른 시장 환경에서 성능이 급격히 저하될 수 있음

:::caution
이 섹션은 ML 앙상블의 **개념적 구조를** 설명하는 것입니다. 실제 ML 구현은 파이썬 기초, 통계학, 머신러닝 이론, 데이터 엔지니어링을 포함하는 별도의 학습 과정이 필요합니다. 위 의사코드를 그대로 운용 시스템에 적용하지 마세요.
:::

## 핵심 요약

- **페어 트레이딩은** 공적분 관계에 있는 두 자산의 스프레드 Z-점수가 ±2 이상 이탈할 때 진입하고, 평균 회귀 시 청산하는 시장 중립 전략입니다.
- 페어 트레이딩의 가장 큰 위험은 공적분 관계 붕괴로, 매월 재검정하고 붕괴 시 즉시 청산하는 규칙이 필수입니다.
- **마켓 메이킹은** 양방향 지정가 주문으로 스프레드 수익을 반복 획득하며, 인벤토리 편향이 쌓이면 호가를 조정하여 리스크를 관리합니다.
- 마켓 메이킹은 초저지연 인프라가 필요하여 개인 투자자보다 전문 기관이 유리하지만, 원리 이해는 호가창 분석에 직접 도움이 됩니다.
- **ML 앙상블은** 여러 모델의 가중 신호를 결합하여 단일 모델의 약점을 보완하지만, 과적합과 데이터 품질이 성패를 좌우합니다.
- ML 트레이딩 시스템에서 룩어헤드 바이어스 방지와 아웃오브샘플 검증은 협상 불가한 필수 요건입니다.
