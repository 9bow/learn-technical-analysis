## 추세추종 전략 개요

추세추종(Trend Following)은 **가격이 한 방향으로 움직이는 추세를** 식별하고, 그 방향으로 포지션을 취하는 전략군입니다. "추세는 친구다(The Trend Is Your Friend)"라는 격언처럼, 시장이 방향을 잡은 후 그 흐름에 올라타 수익을 극대화하고, 추세가 꺾일 때 손절하는 방식으로 작동합니다.

추세추종 전략의 공통 특징:
- 승률은 낮지만(30~45%) 손익비가 높음
- 횡보장에서 잦은 손절 발생
- 큰 추세에서 높은 수익 추구
- 기계적 규칙으로 감정 개입 최소화

## 전략 9: 이동평균 크로스오버 (MA Crossover)

### 전략 개요

이동평균 크로스오버는 **단기 이동평균선이 장기 이동평균선을 교차하는 시점을** 매매 신호로 삼는 가장 고전적인 추세추종 전략입니다. 단기 MA가 장기 MA를 상향 돌파하면 **골든크로스(Golden Cross)**, 하향 돌파하면 **데드크로스(Dead Cross)** 라 부릅니다.

### 매매 공식

$$Signal = \begin{cases} Buy & \text{if } MA_{50} > MA_{200} \\ Sell & \text{if } MA_{50} < MA_{200} \end{cases}$$

단순 이동평균(SMA)과 지수 이동평균(EMA) 모두 사용 가능합니다. EMA는 최근 데이터에 더 많은 가중치를 부여하므로 반응이 빠릅니다.

$$SMA_n = \frac{P_1 + P_2 + \cdots + P_n}{n}$$

$$EMA_t = P_t \times \frac{2}{n+1} + EMA_{t-1} \times \left(1 - \frac{2}{n+1}\right)$$

### 조합별 특성 비교

| 조합 | 반응 속도 | 신호 빈도 | 적합한 시장 |
|------|----------|----------|------------|
| 5일 / 20일 SMA | 빠름 | 높음 (월 2~4회) | 단기 스윙, 암호화폐 |
| 20일 / 60일 SMA | 중간 | 중간 (분기 1~3회) | 주식 스윙 |
| 50일 / 200일 SMA | 느림 | 낮음 (연 1~2회) | 주식 포지션 트레이딩 |
| 12일 / 26일 EMA | 빠름 | 높음 | MACD 기반 시스템 |

50일/200일 골든크로스는 기관 투자자가 참고하는 대표 지표로, 발생 시 대형 추세 전환의 신호로 해석됩니다.

### 매매 규칙

```
[이동평균 크로스오버 매매 규칙]

진입 조건:
  매수: MA_단기 > MA_장기 (전일 MA_단기 <= MA_장기)  ← 크로스 발생 시점
  매도: MA_단기 < MA_장기 (전일 MA_단기 >= MA_장기)

청산 조건:
  롱 청산: 데드크로스 발생 또는 반대 신호

포지션 크기:
  기본: 계좌 자산의 100% (주식)
  레버리지 시장: ATR 기반 포지션 사이징 권장

손절:
  시스템 신호에 의한 청산 (별도 손절 없음)
  또는 ATR × 2 기반 고정 손절
```

### 의사코드

```python
# 이동평균 크로스오버 전략 의사코드

def ma_crossover_strategy(prices, short_period=50, long_period=200):
    ma_short = calculate_sma(prices, short_period)
    ma_long  = calculate_sma(prices, long_period)

    position = None  # None: 미보유, 'long': 매수 포지션

    for i in range(long_period, len(prices)):
        # 골든크로스: 단기가 장기를 상향 돌파
        if ma_short[i] > ma_long[i] and ma_short[i-1] <= ma_long[i-1]:
            if position != 'long':
                buy(prices[i+1])   # 다음 봉 시가에 매수
                position = 'long'

        # 데드크로스: 단기가 장기를 하향 돌파
        elif ma_short[i] < ma_long[i] and ma_short[i-1] >= ma_long[i-1]:
            if position == 'long':
                sell(prices[i+1])  # 다음 봉 시가에 매도
                position = None
```

### 장단점

| 구분 | 내용 |
|------|------|
| 장점 | 규칙이 단순하고 이해하기 쉬움 |
| 장점 | 강한 추세에서 큰 수익 포착 가능 |
| 장점 | 기계적 실행으로 감정 개입 없음 |
| 단점 | 횡보장에서 잦은 손절(Whipsaw) 발생 |
| 단점 | 추세 확인 후 진입하므로 초입 구간 놓침 |
| 단점 | 느린 반응으로 청산 시 수익 반납 과다 |

:::tip
50/200일 크로스오버만 사용할 경우 횡보 구간에서 1년에 손절이 1~2회에 불과하지만, 각 손절이 클 수 있습니다. ADX 필터(전략 12 참조)를 추가하면 횡보 구간 진입을 크게 줄일 수 있습니다.
:::

## 전략 10: 터틀 트레이딩 (Turtle Trading)

### 전략 개요

터틀 트레이딩은 1983년 Richard Dennis가 "트레이딩은 가르칠 수 있는가?"를 증명하기 위해 개발한 **원칙 기반 추세추종 시스템입**니다. Dennis는 일반인 23명을 선발하여 2주간 훈련시킨 후 실제 자금을 맡겼고, 이들은 이후 4년간 연평균 80% 이상의 수익을 달성했습니다. 터틀 트레이딩의 핵심은 수익 극대화를 위한 **피라미딩과** 철저한 **손절 규칙입**니다.

### 핵심 개념: ATR과 유닛(Unit)

**ATR(Average True Range)** 은 시장의 일일 변동성을 나타내며, N이라고도 불립니다.

$$ATR_{20} = \frac{1}{20}\sum_{i=1}^{20} TR_i$$

$$TR_i = \max(High_i - Low_i,\ |High_i - Close_{i-1}|,\ |Low_i - Close_{i-1}|)$$

포지션 크기(유닛)는 계좌 자산 대비 1% 리스크를 기준으로 계산합니다.

$$Unit = \frac{\text{계좌자산} \times 0.01}{ATR_{20}}$$

**예시**: 계좌 자산 1억 원, ATR(N) = 2,000원인 주식

$$Unit = \frac{100{,}000{,}000 \times 0.01}{2{,}000} = 500\text{주}$$

### 진입 시스템

터틀 트레이딩은 두 가지 시스템을 병행합니다.

| 구분 | 시스템 1 | 시스템 2 |
|------|---------|---------|
| 진입 신호 | 20일 최고가 돌파 | 55일 최고가 돌파 |
| 청산 신호 | 10일 최저가 하향 | 20일 최저가 하향 |
| 특징 | 빠른 진입, 잦은 신호 | 느린 진입, 큰 추세 포착 |
| 필터 | 직전 시스템1 신호가 수익 → 진입 생략 | 필터 없음 |

$$\text{진입가(롱)} = \max(P_{t-N+1}, \ldots, P_t) = \text{N일 최고가}$$

$$\text{청산가(롱)} = \min(P_{t-M+1}, \ldots, P_t) = \text{M일 최저가}$$

### 피라미딩 규칙

추세가 확인되면 최대 4개 유닛까지 분할 추가 진입합니다.

```
[피라미딩 진입 규칙]

1유닛: 최초 진입 (20일 고점 돌파)
2유닛: 진입가 + 0.5 × ATR 상승 시 추가
3유닛: 진입가 + 1.0 × ATR 상승 시 추가
4유닛: 진입가 + 1.5 × ATR 상승 시 추가

예시 (ATR = 1,000원):
  1유닛 진입가: 50,000원
  2유닛 진입가: 50,500원 (+500원)
  3유닛 진입가: 51,000원 (+1,000원)
  4유닛 진입가: 51,500원 (+1,500원)
  평균 진입가: 50,750원
```

### 손절 규칙

$$\text{손절가} = \text{진입가} - 2 \times ATR_{20}$$

포지션 전체의 손절은 **계좌 자산의 2%** 이내로 제한합니다. 4유닛까지 피라미딩한 경우 각 유닛의 손절폭이 조정됩니다.

```
[손절 계산 예시]

ATR = 1,000원, 1유닛 = 500주
1유닛 손절가: 50,000 - 2,000 = 48,000원
1유닛 손실액: (50,000 - 48,000) × 500 = 1,000,000원 = 계좌의 1%
```

### 의사코드

```python
# 터틀 트레이딩 시스템 1 의사코드

def turtle_trading(prices, highs, lows, account_value):
    atr = calculate_atr(highs, lows, prices, period=20)
    unit_size = (account_value * 0.01) / atr[-1]

    entry_price = max(highs[-20:])   # 20일 최고가
    exit_price  = min(lows[-10:])    # 10일 최저가
    stop_loss   = entry_price - 2 * atr[-1]

    units = []

    if current_price > entry_price and len(units) == 0:
        units.append({'price': current_price, 'size': unit_size})

    # 피라미딩: 최대 4유닛까지
    elif len(units) < 4:
        next_entry = units[0]['price'] + len(units) * 0.5 * atr[-1]
        if current_price >= next_entry:
            units.append({'price': current_price, 'size': unit_size})

    # 청산 조건
    if current_price < exit_price or current_price < stop_loss:
        close_all_units(units)
        units = []
```

### 장단점

| 구분 | 내용 |
|------|------|
| 장점 | 피라미딩으로 추세 중 수익 극대화 |
| 장점 | 엄격한 손절로 계좌 보호 |
| 장점 | 검증된 실적 기반 시스템 |
| 단점 | 연속 손절 시 계좌 감소(드로다운) 큼 |
| 단점 | 횡보장에서 잦은 신호로 비용 누적 |
| 단점 | 피라미딩 후 추세 반전 시 수익 반납 과다 |

## 전략 11: 슈퍼트렌드 (Supertrend)

### 전략 개요

슈퍼트렌드(Supertrend)는 **ATR을 기반으로 동적으로 움직이는 추세선을** 생성하여, 가격이 해당 선을 돌파할 때 추세 방향 전환을 신호로 삼는 지표입니다. 이동평균선과 달리 변동성에 따라 선폭이 조정되어 횡보장에서의 가짜 신호를 줄이는 효과가 있습니다.

### 계산 공식

기본 밴드(Basic Band)를 먼저 계산합니다.

$$HL_2 = \frac{High + Low}{2}$$

$$\text{Upper Band} = HL_2 + k \times ATR_{n}$$

$$\text{Lower Band} = HL_2 - k \times ATR_{n}$$

슈퍼트렌드는 가격 방향에 따라 밴드를 선택합니다.

$$SuperTrend = \begin{cases} \text{Upper Band} & \text{if 하락 추세 (가격이 Upper Band 아래)} \\ \text{Lower Band} & \text{if 상승 추세 (가격이 Lower Band 위)} \end{cases}$$

기본 파라미터: $n = 10$ (ATR 기간), $k = 3$ (배수)

### 추세 전환 규칙

```
[슈퍼트렌드 신호 생성 규칙]

상태 변수: trend = 'up' | 'down'

초기 상태: price > Upper Band → trend = 'up'
           price < Upper Band → trend = 'down'

매 봉마다:
  if trend == 'down':
    if close > Upper Band:
      trend = 'up'     ← 매수 신호 발생
      SuperTrend = Lower Band

  if trend == 'up':
    if close < Lower Band:
      trend = 'down'   ← 매도 신호 발생
      SuperTrend = Upper Band

밴드 조정 규칙 (추세 유지 시):
  상승 추세: Lower Band = max(Lower Band_t, Lower Band_{t-1})
             → 선이 아래로 내려가지 않아 손절선 역할
  하락 추세: Upper Band = min(Upper Band_t, Upper Band_{t-1})
```

### 파라미터별 특성

| k 값 | ATR 기간 | 민감도 | 적합한 시장 |
|------|---------|--------|------------|
| 1.5 | 7 | 매우 높음 | 암호화폐 단기 |
| 2.0 | 10 | 높음 | 주식 스윙 |
| 3.0 | 10 | 중간 | 주식 포지션 (기본값) |
| 4.0 | 14 | 낮음 | 선물 장기 추세 |

### 의사코드

```python
# 슈퍼트렌드 계산 의사코드

def supertrend(highs, lows, closes, atr_period=10, multiplier=3.0):
    atr = calculate_atr(highs, lows, closes, atr_period)

    supertrend_values = []
    trend = []

    for i in range(atr_period, len(closes)):
        hl2 = (highs[i] + lows[i]) / 2
        upper = hl2 + multiplier * atr[i]
        lower = hl2 - multiplier * atr[i]

        # 밴드 조정 (직전 밴드와 비교)
        if i > atr_period:
            upper = min(upper, prev_upper) if closes[i-1] <= prev_upper else upper
            lower = max(lower, prev_lower) if closes[i-1] >= prev_lower else lower

        # 추세 결정
        if closes[i] > upper:
            current_trend = 'up'
            st_value = lower
        elif closes[i] < lower:
            current_trend = 'down'
            st_value = upper
        else:
            current_trend = prev_trend
            st_value = lower if current_trend == 'up' else upper

        supertrend_values.append(st_value)
        trend.append(current_trend)
        prev_upper, prev_lower, prev_trend = upper, lower, current_trend

    return supertrend_values, trend

# 매매 신호 생성
def generate_signals(closes, trend):
    signals = []
    for i in range(1, len(trend)):
        if trend[i] == 'up' and trend[i-1] == 'down':
            signals.append(('BUY', closes[i+1]))   # 다음 봉 시가
        elif trend[i] == 'down' and trend[i-1] == 'up':
            signals.append(('SELL', closes[i+1]))
    return signals
```

### 장단점

| 구분 | 내용 |
|------|------|
| 장점 | ATR 기반으로 변동성에 자동 적응 |
| 장점 | 신호가 명확하고 재진입 시점이 직관적 |
| 장점 | 상승 추세에서 동적 손절선 역할 |
| 단점 | 횡보 구간에서 빈번한 방향 전환 신호 |
| 단점 | 파라미터(k, n) 조합에 따라 성과 차이 큼 |
| 단점 | 갭 발생 시 손절가와 실제 청산가 차이 발생 |

:::note
슈퍼트렌드는 단독 사용보다 ADX와 결합할 때 효과적입니다. ADX > 25인 구간에서만 슈퍼트렌드 신호를 사용하면 횡보 구간의 가짜 신호를 대폭 줄일 수 있습니다.
:::

## 전략 12: ADX 필터 추세추종

### 전략 개요

ADX(Average Directional Index)는 **추세의 강도를** 0~100 사이 숫자로 측정하는 지표입니다. 방향(상승/하락)이 아닌 강도만 나타내므로, 다른 추세추종 전략의 필터로 사용됩니다. ADX > 25이면 추세가 충분히 강하다고 판단하여 추세추종 진입을 허용하고, ADX < 20이면 횡보 구간으로 판단하여 진입을 보류합니다.

### 계산 공식

방향성 이동(Directional Movement)을 먼저 계산합니다.

$$+DM = \begin{cases} High_t - High_{t-1} & \text{if } High_t - High_{t-1} > Low_{t-1} - Low_t \text{ and } > 0 \\ 0 & \text{otherwise} \end{cases}$$

$$-DM = \begin{cases} Low_{t-1} - Low_t & \text{if } Low_{t-1} - Low_t > High_t - High_{t-1} \text{ and } > 0 \\ 0 & \text{otherwise} \end{cases}$$

방향성 지수(Directional Index)를 계산합니다.

$$+DI_{14} = \frac{EMA_{14}(+DM)}{ATR_{14}} \times 100$$

$$-DI_{14} = \frac{EMA_{14}(-DM)}{ATR_{14}} \times 100$$

ADX는 DI 차이의 이동평균입니다.

$$DX = \frac{|+DI - (-DI)|}{+DI + (-DI)} \times 100$$

$$ADX = EMA_{14}\left(\frac{|+DI - (-DI)|}{+DI + (-DI)} \times 100\right)$$

### ADX 해석 기준

| ADX 값 | 추세 강도 | 전략 적용 |
|--------|---------|---------|
| 0 ~ 20 | 추세 없음 (횡보) | 추세추종 진입 금지 |
| 20 ~ 25 | 약한 추세 시작 | 조심스럽게 진입 가능 |
| 25 ~ 40 | 강한 추세 | 추세추종 진입 적합 |
| 40 ~ 60 | 매우 강한 추세 | 기존 포지션 유지 |
| 60 이상 | 과열 가능성 | 신규 진입 주의 |

### MA 크로스 + ADX 결합 규칙

```
[ADX 필터 추세추종 매매 규칙]

조건 1 (추세 강도 필터):
  ADX > 25  → 추세 구간, 신호 유효
  ADX <= 25 → 횡보 구간, 신호 무시

조건 2 (방향 필터 — DI 교차):
  +DI > -DI  → 상승 추세
  +DI < -DI  → 하락 추세

조건 3 (진입 신호 — MA 크로스오버):
  골든크로스 발생 AND ADX > 25 AND +DI > -DI → 매수
  데드크로스 발생 AND ADX > 25 AND -DI > +DI → 매도

청산:
  MA 데드크로스 발생
  또는 ADX가 25 미만으로 하락 시 포지션 축소 고려
```

### DI 크로스 직접 활용

MA 크로스 없이 DI 크로스만으로도 신호를 생성할 수 있습니다.

```
[DI 크로스 전략]

매수: +DI가 -DI를 상향 돌파 AND ADX > 20
매도: -DI가 +DI를 상향 돌파 AND ADX > 20

주의: ADX는 방향 전환 후 반응이 느려
      진입 후 ADX가 잠시 하락하다가 다시 상승하는 경우가 많음
```

### 의사코드

```python
# ADX 필터 추세추종 의사코드

def adx_trend_strategy(prices, highs, lows,
                       ma_short=20, ma_long=50,
                       adx_period=14, adx_threshold=25):

    ma_s  = calculate_ema(prices, ma_short)
    ma_l  = calculate_ema(prices, ma_long)
    adx, plus_di, minus_di = calculate_adx(highs, lows, prices, adx_period)

    position = None

    for i in range(ma_long, len(prices)):
        trend_strength = adx[i] > adx_threshold
        bullish_direction = plus_di[i] > minus_di[i]
        golden_cross = (ma_s[i] > ma_l[i]) and (ma_s[i-1] <= ma_l[i-1])
        dead_cross   = (ma_s[i] < ma_l[i]) and (ma_s[i-1] >= ma_l[i-1])

        # 매수: 골든크로스 + ADX 조건 + 방향 확인
        if golden_cross and trend_strength and bullish_direction:
            if position != 'long':
                buy(prices[i+1])
                position = 'long'

        # 매도: 데드크로스
        elif dead_cross and position == 'long':
            sell(prices[i+1])
            position = None
```

### 백테스팅 비교: ADX 필터 유무

| 조건 | 연 수익률 | MDD | 거래 횟수 | 승률 |
|------|---------|-----|---------|------|
| MA 크로스만 (5년) | 8.2% | -22.4% | 48회 | 34% |
| MA + ADX 필터 (5년) | 11.5% | -15.1% | 21회 | 47% |

ADX 필터 추가로 거래 횟수는 절반 이하로 줄고, 승률과 MDD가 모두 개선되는 것을 확인할 수 있습니다.

### 장단점

| 구분 | 내용 |
|------|------|
| 장점 | 횡보 구간의 가짜 신호를 효과적으로 제거 |
| 장점 | 거래 비용 절감 (거래 횟수 감소) |
| 장점 | DI 방향으로 추세 방향까지 확인 가능 |
| 단점 | ADX는 후행 지표 — 추세 초기에 반응이 늦음 |
| 단점 | 빠른 추세 전환 구간에서 진입 기회 놓침 |
| 단점 | 파라미터(ADX 기간, 임계값) 최적화 필요 |

:::caution
ADX가 25를 넘은 직후에만 진입하면 추세의 중반 이후 진입하는 경우가 많습니다. 초기 추세를 놓치는 대신 가짜 신호를 줄이는 트레이드오프를 인지하고 사용해야 합니다.
:::

| 전략 | 신호 속도 | 횡보 내성 | 구현 복잡도 | 권장 적용 시장 |
|------|---------|---------|-----------|--------------|

| 슈퍼트렌드 | 빠름 | 중간 | 중간 | 주식, 암호화폐 |

## 핵심 요약

- 추세추종 전략은 **낮은 승률과 높은 손익비의** 조합으로 장기 수익을 추구합니다.
- 이동평균 크로스오버는 가장 단순한 형태로, 50/200일 골든크로스가 대표적입니다.
- 터틀 트레이딩은 ATR 기반 유닛 사이징과 피라미딩으로 추세에서 수익을 극대화합니다.
- 슈퍼트렌드는 ATR 밴드로 동적 추세선을 생성하며, 방향 전환 시 자동으로 신호를 발생시킵니다.
- ADX 필터는 추세 강도를 측정하여 횡보 구간의 가짜 신호를 제거하는 데 효과적입니다.
- 모든 추세추종 전략은 **횡보장에서의 손절 비용을** 감수하는 대가로 큰 추세에서 수익을 추구합니다.
