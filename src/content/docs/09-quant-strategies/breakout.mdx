## 돌파 전략 개요

돌파(Breakout) 전략은 **가격이 특정 레벨(고점, 저점, 레인지)을 초과할 때** 강한 추세가 시작된다는 전제에서 출발합니다. 가격이 오랫동안 갇혀 있던 범위를 벗어날 때 대기 매수/매도 주문이 한꺼번에 체결되면서 급격한 가격 이동이 발생하는 시장 구조를 이용합니다.

돌파 전략의 공통 특징:
- 추세 초기에 진입하여 추세추종보다 더 나은 진입가 확보
- 가짜 돌파(False Breakout)가 전략의 최대 적
- 거래량 또는 변동성 확인을 통해 신뢰도 향상
- 진입 후 즉시 손절선 설정이 핵심

## 전략 13: 돈치안 채널 돌파 (Donchian Channel)

### 전략 개요

돈치안 채널(Donchian Channel)은 **N일간의 최고가와 최저가로** 형성된 채널입니다. Richard Donchian이 개발하였으며, 터틀 트레이딩(전략 10)의 기반 신호로 사용됩니다. 채널 상단 돌파 시 매수, 채널 하단 돌파 시 매도 또는 숏 진입합니다.

### 계산 공식

$$DC_{Upper} = \max(P_{t-N+1},\ P_{t-N+2},\ \ldots,\ P_t)$$

$$DC_{Lower} = \min(P_{t-N+1},\ P_{t-N+2},\ \ldots,\ P_t)$$

$$DC_{Middle} = \frac{DC_{Upper} + DC_{Lower}}{2}$$

여기서 $P_t$는 각 봉의 고가(Upper) 또는 저가(Lower)입니다.

### 시스템1 vs 시스템2

터틀 트레이딩과 동일하게 두 시스템을 병행 사용합니다.

| 구분 | 시스템 1 | 시스템 2 |
|------|---------|---------|
| 채널 기간 | 20일 | 55일 |
| 진입 신호 | 20일 최고가 상향 돌파 | 55일 최고가 상향 돌파 |
| 청산 신호 | 10일 최저가 하향 돌파 | 20일 최저가 하향 돌파 |
| 신호 빈도 | 높음 | 낮음 |
| 추세 포착 | 중기 추세 | 대형 추세 |
| 필터 | 직전 시스템1 신호 수익 시 생략 | 없음 |

### 매매 규칙

```
[돈치안 채널 돌파 매매 규칙]

매수 진입:
  오늘 고가 > DC_Upper(N일) → 다음 봉 시가에 매수

매수 청산:
  오늘 저가 < DC_Lower(M일) → 다음 봉 시가에 청산

손절:
  진입가 - 2 × ATR (터틀 방식)
  또는 DC_Lower(N일) 하향 돌파

포지션 크기:
  Unit = (계좌 × 0.01) / ATR_{20}

주의:
  당일 돌파 봉에서 바로 진입하지 않음
  → 당일 종가 확인 후 다음 봉 시가에 진입 (확인 진입)
  → 단, 갭 발생 시 슬리피지 위험
```

### 가짜 돌파 필터

돈치안 채널 단독 사용 시 가짜 돌파 비율이 높습니다. 다음 필터를 추가합니다.

```
[가짜 돌파 감소 기법]

방법 1: 종가 확인
  고가가 채널을 돌파했더라도 종가가 채널 내부이면 신호 무효

방법 2: ATR 최소 돌파폭
  돌파폭 > 0.5 × ATR 이어야 신호 유효

방법 3: 거래량 확인 (전략 15 참조)
  돌파 당일 거래량 > 20일 평균 거래량 × 1.5

방법 4: 채널 폭 조건
  DC_Upper - DC_Lower > ATR_평균 × 2
  → 채널이 충분히 좁아진 후 돌파만 유효
```

### 의사코드

```python
# 돈치안 채널 돌파 전략 의사코드

def donchian_breakout(highs, lows, closes, n=20, m=10, account_value=10_000_000):
    atr = calculate_atr(highs, lows, closes, period=20)

    position = None
    entry_price = None

    for i in range(n, len(closes)):
        dc_upper = max(highs[i-n:i])   # 오늘 포함 안함 (전일까지 기준)
        dc_lower = min(lows[i-m:i])

        unit_size = (account_value * 0.01) / atr[i]
        stop_loss = dc_upper - 2 * atr[i]  # 진입 후 손절가

        # 매수 진입: 오늘 고가가 채널 상단 돌파
        if highs[i] > dc_upper and position is None:
            # 종가 확인 진입
            if closes[i] > dc_upper:
                entry_price = get_next_open(i)  # 다음 봉 시가
                buy(entry_price, unit_size)
                position = 'long'
                stop_loss = entry_price - 2 * atr[i]

        # 청산: 저가가 청산 채널 하단 돌파
        elif position == 'long':
            if lows[i] < dc_lower or lows[i] < stop_loss:
                sell(get_next_open(i))
                position = None

    return trade_log
```

### 장단점

| 구분 | 내용 |
|------|------|
| 장점 | 규칙이 명확하고 자동화 구현이 간단 |
| 장점 | 대형 추세의 초입을 포착 가능 |
| 장점 | 터틀 트레이딩과 결합 시 강력한 시스템 완성 |
| 단점 | 가짜 돌파 비율이 높아 연속 손절 발생 가능 |
| 단점 | 횡보장에서 채널 상/하단이 지속 교차되어 손실 누적 |
| 단점 | 긴 채널(55일) 사용 시 대형 추세 초반 상당 부분 놓침 |

## 전략 14: ATR 변동성 돌파 (Larry Williams)

### 전략 개요

ATR 변동성 돌파 전략은 Larry Williams가 개발한 **당일 내 돌파** 시스템입니다. 오늘 시가에 전일 ATR의 일정 비율(K값)을 더한 가격을 돌파하면 매수 진입하고, **당일 장 마감 시 무조건 청산합**니다. 오버나이트 리스크를 제거하면서 일중 추세를 포착하는 데 특화되어 있습니다.

### 계산 공식

$$\text{진입가} = \text{금일 시가} + ATR_{N} \times K$$

$$\text{청산가} = \text{당일 종가} \text{ (장 마감 시 무조건 청산)}$$

$$ATR_N = \frac{1}{N}\sum_{i=1}^{N} TR_i$$

여기서 $K$는 최적화 파라미터(보통 0.4~0.6)입니다.

### K값 최적화

K값은 시장과 종목에 따라 최적값이 다릅니다.

| K값 | 진입 빈도 | 특성 | 권장 시장 |
|-----|---------|------|---------|
| 0.2 | 매우 높음 | 민감, 가짜 신호 많음 | 부적합 |
| 0.4 | 높음 | 빠른 진입, 적당한 신호 | 변동성 낮은 종목 |
| 0.5 | 중간 | 균형 (Larry Williams 기본값) | 대부분 시장 |
| 0.6 | 낮음 | 강한 돌파만 포착 | 변동성 높은 시장 |
| 0.8 | 매우 낮음 | 극단적 돌파만 | 암호화폐 고변동 |

K값 최적화 방법:

```
[K값 최적화 절차]

1. K = 0.1 ~ 1.0 범위에서 0.05 단위로 백테스팅
2. 각 K값의 연 수익률, 샤프 비율, MDD 기록
3. 최적 K값 ± 0.1 범위에서도 안정적 성과 확인 (강건성 검증)
4. 파라미터 민감도가 낮은 K값 선택 (과최적화 방지)

주의: 인샘플에서 최적 K값을 찾은 후
      아웃오브샘플로 반드시 검증
```

### 매매 규칙

```
[ATR 변동성 돌파 매매 규칙]

매수 조건:
  장중 가격이 (시가 + ATR_{N} × K)를 돌파 → 즉시 매수

청산 조건:
  당일 종가 시점에 무조건 청산 (오버나이트 금지)
  또는 손절가 (시가 - ATR_{N} × 0.5) 하향 시 장중 청산

포지션 크기:
  계좌 자산의 1~5% 리스크 기준
  = (계좌 × 리스크율) / (ATR × K)

예시 계산:
  시가: 50,000원
  ATR(20): 1,000원
  K: 0.5
  진입가: 50,000 + 1,000 × 0.5 = 50,500원
  손절가: 50,000 - 1,000 × 0.5 = 49,500원 (1,000원 리스크)
```

### 하락 방어 버전

원래 전략은 롱 전용이지만, 하락 버전도 대칭적으로 적용할 수 있습니다.

$$\text{숏 진입가} = \text{금일 시가} - ATR_{N} \times K$$

```
[양방향 ATR 돌파 규칙]

상방 돌파: 시가 + ATR × K 돌파 → 매수 진입
하방 돌파: 시가 - ATR × K 하향 → 공매도 진입

단, 같은 날 양방향 신호 발생 시 더 강한 쪽만 선택
또는 먼저 발생한 신호만 사용
```

### 의사코드

```python
# ATR 변동성 돌파 전략 의사코드 (일봉 기준)

def atr_volatility_breakout(opens, highs, lows, closes,
                             atr_period=20, K=0.5, risk_pct=0.02):
    atr = calculate_atr(highs, lows, closes, atr_period)

    trade_log = []
    account_value = 10_000_000  # 초기 자본

    for i in range(atr_period + 1, len(opens)):
        today_open = opens[i]
        prev_atr   = atr[i-1]  # 전일 ATR 사용

        buy_trigger  = today_open + prev_atr * K
        stop_trigger = today_open - prev_atr * 0.5

        # 포지션 크기 계산
        risk_amount = account_value * risk_pct
        position_size = risk_amount / (prev_atr * K)

        bought = False
        entry_price = None

        # 장중 틱 데이터로 돌파 확인 (일봉 시뮬레이션)
        if highs[i] >= buy_trigger:
            entry_price = buy_trigger
            bought = True

        if bought:
            # 손절 확인
            if lows[i] < stop_trigger and stop_trigger < entry_price:
                exit_price = stop_trigger
            else:
                exit_price = closes[i]  # 종가 청산

            pnl = (exit_price - entry_price) * position_size
            account_value += pnl
            trade_log.append({
                'date': i,
                'entry': entry_price,
                'exit': exit_price,
                'pnl': pnl
            })

    return trade_log, account_value
```

### 장단점

| 구분 | 내용 |
|------|------|
| 장점 | 오버나이트 리스크 없음 (당일 청산) |
| 장점 | 강한 일중 모멘텀을 체계적으로 포착 |
| 장점 | K값 하나만 최적화하면 되어 과최적화 위험 낮음 |
| 단점 | 당일 장 마감 후 연속 상승 추세 이익 포기 |
| 단점 | 갭 상승 후 시가가 이미 고점인 경우 불리 |
| 단점 | 일봉 기준이므로 빈번한 진입 불가 (하루 1회) |

:::tip
Larry Williams는 이 전략을 코스피, S&P500, 원자재 선물에서 모두 테스트하여 강건성을 확인했습니다. K=0.5를 기본값으로 시작하되, 암호화폐처럼 변동성이 극단적인 시장에서는 K=0.6~0.7이 더 적합합니다.
:::

## 전략 15: 거래량 확인 돌파

### 전략 개요

가격 돌파만으로는 가짜 신호를 걸러내기 어렵습니다. 거래량 확인 돌파 전략은 **가격 돌파와 거래량 급증이 동시에 발생할** 때만 진입하여 신뢰도를 높입니다. 기관 투자자나 대형 세력의 참여 여부를 거래량으로 확인하는 원리입니다.

### 신호 조건 공식

$$\text{신호} = \text{고점 돌파} \cap V_t > 1.5 \times \bar{V}_{20}$$

여기서:
- $\text{고점 돌파}$: 현재 가격이 N일 최고가를 초과
- $V_t$: 돌파 발생 봉의 거래량
- $\bar{V}_{20}$: 20일 평균 거래량
- $1.5$: 거래량 기준 배수 (1.5배 이상)

### 거래량 기준 배수 선택

| 기준 배수 | 신호 빈도 | 신뢰도 | 특성 |
|---------|---------|--------|------|
| 1.2배 | 높음 | 낮음 | 진입 기회 많으나 가짜 신호 포함 |
| 1.5배 | 중간 | 중간 | 균형적 기준 (기본값) |
| 2.0배 | 낮음 | 높음 | 강한 돌파만 포착, 놓치는 경우도 많음 |
| 3.0배 | 매우 낮음 | 매우 높음 | 뉴스/이벤트 수반 폭발적 돌파만 |

### 추가 필터 조합

거래량 확인을 다른 조건과 결합할 수 있습니다.

```
[거래량 확인 돌파 결합 규칙]

기본 신호:
  가격 > N일 최고가 AND 거래량 > 1.5 × MA_20(거래량)

강화 조건 (선택):
  AND 종가가 당일 레인지 상위 70% 이상 위치
      (Close - Low) / (High - Low) > 0.7
      → 강한 매수 압력 확인

  AND 가격이 채널 돌파 후 당일 중 되돌아오지 않음
      → 장 마감 시 종가가 여전히 채널 상단 위

  AND 거래량 증가가 3일 이상 연속 증가 추세
      → 지속적 관심도 증가 확인
```

### OBV와의 연계

거래량 돌파 신호의 신뢰도를 높이려면 **OBV(On-Balance Volume)** 가 가격보다 먼저 신고점을 갱신하는지 확인합니다.

$$OBV_t = \begin{cases} OBV_{t-1} + V_t & \text{if } Close_t > Close_{t-1} \\ OBV_{t-1} - V_t & \text{if } Close_t < Close_{t-1} \\ OBV_{t-1} & \text{if } Close_t = Close_{t-1} \end{cases}$$

```
[OBV 선행 확인 규칙]

신뢰도 높은 돌파:
  OBV가 이미 전고점 갱신 → 가격 돌파 → 강력한 매수 신호

신뢰도 낮은 돌파:
  OBV가 전고점 아래 → 가격 돌파 → 가짜 돌파 가능성 높음
  (세력 없는 상승 → 곧 되돌림 가능)
```

### 의사코드

```python
# 거래량 확인 돌파 전략 의사코드

def volume_confirmed_breakout(highs, lows, closes, volumes,
                               price_period=20, vol_period=20,
                               vol_multiplier=1.5, account_value=10_000_000):

    obv = calculate_obv(closes, volumes)
    atr = calculate_atr(highs, lows, closes, period=14)

    position = None

    for i in range(price_period, len(closes)):
        channel_upper = max(highs[i-price_period:i])
        vol_avg = sum(volumes[i-vol_period:i]) / vol_period

        # 기본 신호: 가격 돌파 + 거래량 급증
        price_breakout  = closes[i] > channel_upper
        volume_surge    = volumes[i] > vol_avg * vol_multiplier

        # 강화 조건: 종가 위치 (레인지 상위 70%)
        candle_range = highs[i] - lows[i]
        strong_close = ((closes[i] - lows[i]) / candle_range) > 0.7 if candle_range > 0 else False

        # OBV 확인: OBV가 이전 20일 고점 이상
        obv_high = max(obv[i-price_period:i])
        obv_confirm = obv[i] >= obv_high

        if price_breakout and volume_surge and strong_close and obv_confirm:
            if position is None:
                entry = get_next_open(i)
                stop  = entry - 2 * atr[i]
                unit  = (account_value * 0.02) / (2 * atr[i])
                buy(entry, unit)
                position = {'entry': entry, 'stop': stop, 'size': unit}

        elif position is not None:
            # 청산: 손절 또는 N일 저점 하향
            dc_lower = min(lows[i-10:i])
            if closes[i] < position['stop'] or closes[i] < dc_lower:
                sell(get_next_open(i), position['size'])
                position = None

    return trade_log
```

| 구분 | 내용 |

| 장점 | 가짜 돌파 제거 효과 탁월 |
| 장점 | 기관 참여 가능성이 높은 강한 신호만 선별 |
| 장점 | OBV 연계로 선행 신호 확인 가능 |
| 단점 | 신호 발생 빈도가 낮아 기회 부족 |
| 단점 | 거래량 데이터가 없는 선물/외환 시장에서는 적용 어려움 |
| 단점 | 기준 배수(1.5배) 설정이 주관적 |

## 전략 16: 오픈 레인지 브레이크아웃 (ORB)

### 전략 개요

오픈 레인지 브레이크아웃(Open Range Breakout, ORB)은 **장 시작 후 N분 동안 형성된 레인지(고가-저가)** 를 기준으로, 이 범위를 벗어나는 방향으로 진입하는 일중 전략입니다. 장 초반에 형성되는 레인지는 당일 시장 참여자들의 기대와 균형이 반영된 구간으로, 이를 돌파할 때 강한 방향성이 나타나는 경향이 있습니다.

### 핵심 공식

$$\text{Range} = High_{ORB} - Low_{ORB}$$

$$\text{상단 목표가(Target)} = High_{ORB} + \text{Range}$$

$$\text{하단 목표가(Target)} = Low_{ORB} - \text{Range}$$

$$\text{매수 손절가} = Low_{ORB}$$

$$\text{매도 손절가} = High_{ORB}$$

### 15분 vs 30분 ORB 비교

| 구분 | 15분 ORB | 30분 ORB |
|------|---------|---------|
| 레인지 형성 시간 | 장 시작 후 15분 | 장 시작 후 30분 |
| 레인지 크기 | 상대적으로 작음 | 상대적으로 큼 |
| 신호 빈도 | 높음 | 낮음 |
| 가짜 돌파 비율 | 높음 | 낮음 |
| 손절폭 | 작음 | 큼 |
| 권장 시장 | 유동성 매우 높은 종목 | 일반 주식 |
| 손익비 | 1:1 (작은 레인지) | 1:1.5~2 |

한국 주식 시장 기준:
- **15분 ORB**: 09:00~09:15 레인지
- **30분 ORB**: 09:00~09:30 레인지
- **60분 ORB**: 09:00~10:00 레인지 (더 보수적)

### 매매 규칙

```
[ORB 매매 규칙]

레인지 형성:
  09:00 ~ 09:30 (30분 ORB 기준)
  ORB_High = 해당 구간의 최고가
  ORB_Low  = 해당 구간의 최저가
  Range    = ORB_High - ORB_Low

매수 진입:
  09:30 이후 가격이 ORB_High를 상향 돌파
  → 즉시 시장가 매수
  목표가: ORB_High + Range (1:1 손익비)
  손절가: ORB_Low

공매도 진입:
  09:30 이후 가격이 ORB_Low를 하향 돌파
  → 즉시 시장가 공매도 (공매도 가능 종목에서)
  목표가: ORB_Low - Range
  손절가: ORB_High

강제 청산:
  장 마감 30분 전 (14:30 한국 주식 기준) 무조건 청산
  → 오버나이트 리스크 제거
```

### 신뢰도 향상 필터

```
[ORB 필터 목록]

필터 1: 레인지 크기 확인
  Range < ATR × 0.3 → 레인지가 너무 좁음, 신호 무효
  Range > ATR × 2.0 → 레인지가 너무 넓음, 손절폭 과대

필터 2: 돌파 강도 확인
  돌파 봉의 크기 > Range × 0.5 → 강한 돌파
  돌파 봉의 크기 < Range × 0.2 → 약한 돌파, 가짜 신호 가능성

필터 3: 갭 조건
  전일 대비 갭 없이 시작한 날 → 레인지 유효성 높음
  갭 상승 또는 갭 하락 시작 → ORB 방향이 왜곡될 수 있음

필터 4: 전일 추세 방향
  전일 종가 > 5일 MA → 상방 돌파 신호 선호
  전일 종가 < 5일 MA → 하방 돌파 신호 선호
  (추세 방향과 일치하는 돌파만 진입)
```

### 목표가 확장

초기 목표가(1:1 손익비) 도달 후 포지션 일부를 유지하여 추세를 더 끌고 갈 수 있습니다.

```
[ORB 확장 목표가 설정]

1차 목표: ORB_High + Range × 1.0 (50% 청산)
2차 목표: ORB_High + Range × 1.5 (30% 청산)
3차 목표: ORB_High + Range × 2.0 (20% 청산)

1차 목표 도달 후:
  손절선을 ORB_High (진입 근처)로 이동 → 리스크 프리(손실 없는 상태)

2차 목표 도달 후:
  손절선을 1차 목표가로 이동 → 최소 수익 확보
```

### 의사코드

```python
# 오픈 레인지 브레이크아웃 의사코드 (분봉 기준)

def orb_strategy(candles, orb_minutes=30, account_value=10_000_000):
    """
    candles: 분봉 데이터 리스트 [{time, open, high, low, close, volume}]
    """

    daily_groups = group_by_day(candles)

    trade_log = []

    for day, day_candles in daily_groups.items():

        # 1. ORB 레인지 형성 (첫 N분)
        orb_candles = [c for c in day_candles if c['time'] < market_open + orb_minutes]
        orb_high = max(c['high'] for c in orb_candles)
        orb_low  = min(c['low']  for c in orb_candles)
        orb_range = orb_high - orb_low

        atr_day = calculate_atr_daily(day)

        # 레인지 크기 필터
        if orb_range < atr_day * 0.3 or orb_range > atr_day * 2.0:
            continue  # 이날 ORB 신호 무시

        target_buy  = orb_high + orb_range
        target_sell = orb_low  - orb_range

        position = None
        entry_price = None

        # 2. ORB 이후 매매 구간
        post_orb = [c for c in day_candles if c['time'] >= market_open + orb_minutes]

        for candle in post_orb:

            # 강제 청산: 마감 30분 전
            if candle['time'] >= market_close - 30:
                if position is not None:
                    sell(candle['open'])
                    position = None
                break

            if position is None:
                # 상방 돌파
                if candle['high'] > orb_high:
                    entry_price  = orb_high
                    stop_price   = orb_low
                    risk         = entry_price - stop_price
                    unit_size    = (account_value * 0.02) / risk
                    buy(entry_price, unit_size)
                    position = 'long'

                # 하방 돌파
                elif candle['low'] < orb_low:
                    entry_price  = orb_low
                    stop_price   = orb_high
                    risk         = stop_price - entry_price
                    unit_size    = (account_value * 0.02) / risk
                    short(entry_price, unit_size)
                    position = 'short'

            elif position == 'long':
                if candle['low'] <= stop_price:
                    sell(stop_price)
                    position = None
                elif candle['high'] >= target_buy:
                    sell(target_buy)
                    position = None

            elif position == 'short':
                if candle['high'] >= stop_price:
                    buy_cover(stop_price)
                    position = None
                elif candle['low'] <= target_sell:
                    buy_cover(target_sell)
                    position = None

    return trade_log
```

### 시장별 ORB 적용 시간

| 시장 | 레인지 시간 | 추천 ORB |
|------|-----------|---------|
| 한국 주식 (KRX) | 09:00 ~ 09:30 | 30분 ORB |
| 미국 주식 (NYSE/NASDAQ) | 09:30 ~ 10:00 | 30분 ORB |
| 암호화폐 (24시간) | 00:00 ~ 01:00 | 1시간 ORB (아시아 세션 기준) |
| 선물 (CME) | 개장 후 첫 30분 | 30분 ORB |

### 장단점

| 구분 | 내용 |
|------|------|
| 장점 | 오버나이트 리스크 없음 |
| 장점 | 손절가와 목표가가 레인지 기준으로 명확히 정의됨 |
| 장점 | 하루 1~2회 거래로 단순한 일과 |
| 단점 | 장 초반 변동성이 높아 가짜 돌파 발생 잦음 |
| 단점 | 시장이 레인지 내에서 종일 머물면 수익 없음 |
| 단점 | 분봉 데이터 필요 — 일봉만 사용하는 시스템에서 구현 불가 |

:::caution
ORB는 유동성이 충분한 대형 종목이나 지수 ETF에서 가장 효과적입니다. 소형주나 거래량이 적은 종목에서는 가짜 돌파와 슬리피지 문제로 전략이 제대로 작동하지 않습니다.
:::

### 결합 사용 권장

```
[돌파 전략 결합 예시]

1단계: 돈치안 채널(55일)로 대형 추세 방향 확인
  → DC 상단 위: 상방 추세 확인

2단계: ATR 변동성 돌파로 매일 진입 신호 생성
  → 시가 + ATR × 0.5 돌파 시 매수

3단계: 거래량 확인으로 신호 필터링
  → 거래량 > 20일 평균 × 1.5 시에만 진입
```

## 핵심 요약

- 돌파 전략은 **가격이 레인지를 벗어나는 시점의 강한 방향성을** 포착합니다.
- 돈치안 채널은 N일 고점/저점을 기준으로 하는 고전적 돌파 시스템으로, 터틀 트레이딩의 기반입니다.
- ATR 변동성 돌파는 당일 오전 돌파 후 장 마감 청산하는 Larry Williams의 시스템으로, 오버나이트 리스크를 피합니다.
- 거래량 확인 돌파는 가격 돌파에 거래량 조건을 추가하여 가짜 돌파를 효과적으로 제거합니다.
- ORB는 장 시작 N분의 레인지를 기준으로 당일 방향성을 결정하는 일중 전략입니다.
- 모든 돌파 전략은 **손절가를 레인지 기준으로 명확히 설정하**는 것이 핵심입니다.
